Fase 1: Múltiplas Janelas

Saindo da tela única

Slide 03

Por que criar outra janela?

Olá equipe! Até agora, o nosso App tem apenas uma tela principal. Esprememos o formulário de cadastro na barra lateral (Sidebar) para caber tudo no mesmo espaço.

Pensem num Jogo de Videogame

Imaginem jogar um game onde a tela de "Configurações" ou o "Inventário" ficam sempre abertas tampando metade da sua visão. Fica confuso, certo?

No Electron é igual. É melhor abrir uma janela separada (um pop-up) só para o cadastro, deixando o catálogo principal limpo e organizado.

Slide 04

A Fábrica de Janelas

Lembram do comando new BrowserWindow() que usamos no index.js na primeira aula? Ele é uma fábrica. Podemos chamá-lo quantas vezes quisermos para criar telas novas!

index.js (Cozinha / Back-end)
// Quando recebermos o aviso para abrir a tela de cadastro:
ipcMain.on('abrir-cadastro', () => {
    
    // 1. Fabricamos uma SEGUNDA janela menor
    const janelaCadastro = new BrowserWindow({
        width: 400, 
        height: 500,
        webPreferences: { preload: path.join(__dirname, 'preload.js') }
    });

    // 2. Carregamos um arquivo HTML DIFERENTE só para ela
    janelaCadastro.loadFile(path.join(__dirname, '../renderer/cadastro.html'));
    
});
Slide 05

O Bug da Janela Solta

Se nós apenas criarmos a janela nova, ela fica "solta". O usuário pode clicar na janela principal atrás dela, e a janela de cadastro vai se esconder lá no fundo.

Janela Principal
(Ficou por cima sem querer!)
Cadastro
(Escondida)

Isso confunde o usuário e causa bugs! Precisamos conectá-las.

Slide 06

A Solução: Regra de Parentesco

Para resolver, usamos duas palavras mágicas na configuração da nossa nova janela:

1. parent: janelaPrincipal

Avisa o sistema que a janela nova é filha da janela principal.

2. modal: true

Funciona como um "Pause" no jogo. Trava a janela principal inteira. O usuário é obrigado a preencher o formulário ou fechar a janela menor para voltar a usar o App.

Configuração Correta
const janelaModal = new BrowserWindow({
    width: 400, 
    height: 500,
    parent: janelaPrincipal, // Define quem é a MÃE
    modal: true              // TRAVA a tela de trás
});

Fase 2: As Rotas do Electron

O Mapa da Comunicação (IPC)

Slide 08

O Trânsito do App: Resumo Visual

Antes de avançarmos, vamos desenhar o mapa exato de como a Interface (HTML) e o Cérebro (Node.js) conversam entre si.

1
Front-end (HTML / UI)
AÇÃO (Ida)
Back-end (Node.js / Main)
2
Front-end (HTML / UI)
CONSULTA (Bate-e-Volta)
Back-end (Node.js / Main)
3
Front-end (HTML / UI)
AVISO (Volta)
Back-end (Node.js / Main)
Slide 09

Entendendo cada Rota de Código

  1. AÇÃO: ipcMain.on
    O Front-end envia uma ordem para o Backend. É como enviar uma carta: "Node, abre a janela!". O Front não precisa ficar esperando resposta. Ele dispara a ordem e segue em frente.
  2. CONSULTA: ipcMain.handle (e invoke)
    O Front-end faz um pedido e espera (usando await). É como pedir no iFood: "Node, salva os dados?". O Backend recebe o pedido, processa, e devolve a resposta de confirmação (usando return).
  3. AVISO: webContents.send (O Novo!)
    O Backend avisa o Front-end de surpresa. É como receber uma notificação no celular: "Atenção Front, atualiza a tela aí!".
Slide 10

O Enigma da Atualização

Vamos aplicar a ROTA 3 (Aviso) no nosso problema do Pop-up de Cadastro:

  1. A Janela Filha (Cadastro) usa a Rota 2 (Consulta/Invoke) para pedir ao Node para salvar os dados e fica à espera.
  2. O Node.js salva no banco de dados e devolve uma resposta de sucesso (True). A janela filha recebe a confirmação e só então se fecha.
  3. O PROBLEMA: A Janela Principal não sabe que isso aconteceu! Ela continua mostrando a lista antiga de produtos!

A Solução de Arquitetura:

Nós precisamos que o Node.js (Back-end) utilize a Rota 3 para mandar um "Alerta de Celular" para a Janela Principal dizendo: "Atenção: atualiza a tela agora!".

Slide 11

O Código do Rádio Inverso: webContents.send

Veja como o Node e o Front-end configuram esta Rota 3 de "Avisos":

No Back-end (Node)

// O Node grita para a janela!
janelaPrincipal.webContents.send('atualizar-tela');

No Front-end (Garçom)

// Quando ouvir 'atualizar-tela'
onAtualizar: (minhaFuncao) => {
    ipcRenderer.on('atualizar-tela', minhaFuncao);
}

Mãos no Teclado: Construindo nosso Pop-up juntos

Slide 13

Nosso Plano de Ação

Abram o código da aula passada (O Dashboard). Vamos alterá-lo passo a passo, no seu tempo. Se precisar, peça pausa.

  1. Ajustar o index.js (Tornar a janela global).
  2. Modificar o index.html (Trocar o form antigo por um Botão).
  3. Configurar o preload.js (Os 2 novos canais).
  4. Criar a Janela Filha no Node.
  5. Criar a interface da Janela Filha: modal.html.
  6. Criar a lógica para salvar com Rota 2: modal.js.
  7. O Grande Final: Fechar o ciclo usando as Rotas 2 e 3.
Passo 1

Tornando a Janela Global

No seu index.js (Main), a variável janela nasce e morre trancada dentro da função. Precisamos tirá-la de lá para podermos enviar os avisos depois.

src/main/index.js
const { app, BrowserWindow, ipcMain } = require('electron');
const path = require('path');

// 1. CRIAMOS A VARIÁVEL AQUI FORA (Global)
let janelaPrincipal; 

const db = require('../../dados.json');

app.whenReady().then(() => {
    // 2. ATRIBUÍMOS O VALOR A ELA AQUI DENTRO (Sem o 'const')
    janelaPrincipal = new BrowserWindow({
        width: 1000, height: 700,
        webPreferences: { preload: path.join(__dirname, 'preload.js') }
    });
    
    janelaPrincipal.loadFile(path.join(__dirname, '../renderer/index.html'));
    
    // ... restante do código continua igual ...
});

Passo 3

Ensinando Novos Truques ao Garçom

Abram o preload.js. Vamos adicionar o pedido para abrir a janela e o "ouvido" biônico para escutar a ordem de atualizar a tela.

src/main/preload.js
const { contextBridge, ipcRenderer } = require('electron');

contextBridge.exposeInMainWorld('api', {
    
    pedirProdutos: () => {
        return ipcRenderer.invoke('pedir-produtos');
    },

    // NOVO: Rota 1 (Ação) para abrir a janela
    abrirModalCadastro: () => {
        ipcRenderer.send('abrir-modal');
    },

    // NOVO: Rota 3 (Ouvinte) Escuta o Main e roda uma função
    onAtualizarTela: (funcaoCallback) => {
        ipcRenderer.on('atualizar-tela', funcaoCallback);
    }
});
Passo 4

Construindo a Janela Modal

Volte ao index.js (Onde fica o Node). No final de tudo, crie a regra de fabricação da Janela Filha.

src/main/index.js (Adicione no final)
ipcMain.on('abrir-modal', () => {
    
    const janelaModal = new BrowserWindow({
        width: 400, height: 500,
        
        parent: janelaPrincipal, // ATENÇÃO: Define a MÃE
        modal: true,             // ATENÇÃO: TRAVA a tela de trás
        
        // Vamos usar o mesmo garçom (preload) para facilitar
        webPreferences: { preload: path.join(__dirname, 'preload.js') } 
    });

    janelaModal.loadFile(path.join(__dirname, '../renderer/modal.html'));
});

Passo 6

A Lógica do Modal

Crie o modal.js. O papel dele é extrair os dados, enviar a Ordem (Rota 2) para o Node salvar e esperar a confirmação para se fechar.

src/renderer/modal.js
document.querySelector("#formProduto").addEventListener("submit", async (e) => {
    e.preventDefault(); // Evita o piscar da tela
    
    // Captura Mágica (FormData)
    const dados = Object.fromEntries(new FormData(e.target));
    dados.preco = Number(dados.preco);
    dados.estoque = Number(dados.estoque);

    // O Garçom envia para o Node e ESPERA a confirmação (Rota 2)
    const salvouComSucesso = await window.api.salvarProduto(dados);
    
    // Só fecha a janela se o Node confirmar que salvou!
    if (salvouComSucesso) {
        window.close();
    }
});

Adicione no seu preload.js:
salvarProduto: (dados) => ipcRenderer.invoke('salvar', dados)

Passo 7

O Grande Final (Fechando o Ciclo)

O Node recebe o dado, salva, avisa a janela principal para redesenhar a tela (Rota 3) e envia a resposta de sucesso (Rota 2).

index.js (Adicione no final)
ipcMain.handle('salvar', (e, dados) => {
    // 1. Salva no banco de dados (array)
    db.push(dados);

    // 2. ROTA 3! Grita para a tela principal
    janelaPrincipal.webContents.send('atualizar-tela');

    // 3. ROTA 2! Devolve a confirmação de sucesso
    return true; 
});
renderer.js (Principal)
// Usamos o "ouvido" que criamos no preload
window.api.onAtualizarTela(() => {
    console.log("Ouvi o chamado do Node!");
    
    // Volta a executar a função que limpa e desenha a tela
    // (Ela que possui o invoke para pedir a lista atualizada)
    carregarTela();
});

Missões Extras (XP Adicional)

Trabalhar com pop-ups é muito interessante. Vamos testar os limites do nosso sistema!

  1. Modal Elegante: Altere a configuração da sua BrowserWindow filha (Modal) no index.js para remover a barra superior do Windows (use a opção frame: false) e remova a opção de redimensionar a janela (resizable: false).
  2. Botão Cancelar: Como removeu o "X" de fechar janela na missão 1, crie um botão "Cancelar" no seu modal.html (com estilo vermelho). Faça com que ele use o window.close() ou envie uma Ordem (Rota 1) para o Main forçar o fechamento da janela Modal sem salvar nada.
  3. Janela Sobre: Na Topbar da Janela Principal, adicione um botão "Info". Ao clicar, o sistema deve abrir uma janela pequena (sem ser Modal) contendo o seu nome e versão do sistema.
  4. Prevenção de Bugs: Se clicar no botão "Info" da missão anterior várias vezes seguidas, ele vai abrir 10 janelas iguais! Pesquise no Google ou peça ajuda à IA: "No Electron, como o Node pode verificar se uma janela secundária já está aberta e impedir que ela abra novamente no mesmo clique?".
