1. A Função Clássica

Em C, vocês declaravam tipos de retorno e de argumentos. No JS clássico, é mais solto, mas a estrutura visual é similar.

Linguagem C

int somar(int a, int b) {
    return a + b;
}

Javascript (Function)

function somar(a, b) {
    return a + b;
}

Note a ausência de int. O JS descobre o tipo na hora.

2. Arrow Functions (Funções Flecha)

No JS moderno, funções são tratadas como Valores. Podemos guardar uma função dentro de uma variável, assim como guardamos um número.

const somar = (a, b) => {
    return a + b;
};

Por que usamos const?

  • Para garantir que a função somar não seja sobrescrita por outra coisa (como um número) no meio do código.
  • É uma boa prática de segurança.

3. O Poder do Retorno Implícito

Aqui o JS brilha sobre o C. Se sua função apenas calcula e retorna um valor (uma linha), podemos remover:

  1. A palavra return.
  2. As chaves {}.
const dobro = (n) => {
  return n * 2;
};

4. Parâmetros Padrão (Default)

Em C, para ter comportamentos diferentes, fazíamos sobrecarga (várias funções com mesmo nome). Em JS, definimos o valor padrão no argumento.

// Se a cor não for enviada, assume "Azul"
const criarBotao = (texto, cor = "Azul") => {
    console.log(`Botão ${texto} na cor ${cor}`);
};

criarBotao("Salvar", "Vermelho"); // Cor: Vermelho
criarBotao("Cancelar");           // Cor: Azul (Automático!)
10

5. Parâmetros Rest (...)

E se quisermos uma função que aceita qualquer quantidade de números? Em C usaríamos vetores complexos. Em JS, usamos o Rest Operator ....

// ...notas transforma todos os argumentos num array
const mostrarNotas = (...notas) => {
    console.log(notas); // [10, 5, 8, 9...]
};

mostrarNotas(10, 20, 30, 40, 50);

Muito útil para somatórios ou logs.

6. Short-Circuit (Curto-Circuito)

Podemos usar || (OU) e && (E) para tomar decisões sem usar if.

Operador || (OU)

Usa o primeiro valor verdadeiro (valor padrão).

let usuario = ""; // Vazio é falso
let nome = usuario || "Desconhecido";
console.log(nome); // "Desconhecido"

Operador && (E)

Só executa a segunda parte se a primeira for verdadeira.

let logado = true;
logado && console.log("Painel Admin");
// Equivalente a: if(logado) {...}

7. Nullish Coalescing (??)

O problema do || é que ele considera o número 0 como falso. Mas as vezes 0 é um valor válido (ex: nota 0).

O ?? só troca se for null ou undefined.

let nota = 0; // Aluno tirou zero

// Com OU (Errado)
console.log(nota || 10); // Imprime 10! (Opa!)

// Com Nullish (Certo)
console.log(nota ?? 10); // Imprime 0.

8. Optional Chaining (?.)

Comparação com C: Em C, se você tem um ponteiro de struct aluno->endereco->rua e endereco é NULL, o programa fecha (Segmentation Fault).

No JS, também dá erro. Mas o ?. resolve isso.

const usuario = { nome: "Will", endereco: null };

// Erro Crítico
// console.log(usuario.endereco.rua); 

// Modo Seguro (Retorna undefined, não trava)
console.log(usuario?.endereco?.rua);

9. Estruturas de Repetição

Temos os clássicos do C e os modernos do JS.

Os Clássicos (Igual C)

while (condicao) { ... }

do { ... } while (condicao);

for (let i=0; i<10; i++) { ... }

O Moderno (for...of)

Ideal para percorrer arrays e strings.

for (const item of lista) {
    console.log(item);
}
