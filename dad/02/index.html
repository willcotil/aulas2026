Parte 1: Os Clássicos

While, Do-While e For

03

O Loop while

Idêntico ao C. Executa enquanto a condição for verdadeira. O teste é feito no início.

let energia = 5;

while (energia > 0) {
    console.log(`Energia: ${energia}`);
    energia--; // Decremento
}
console.log("Sem energia.");
04

O Loop do...while

Garante que o bloco rode pelo menos uma vez, pois o teste é feito no final.

let resposta = "";

do {
    console.log("Executando menu...");
    // Suponha que o usuário digitou "S"
    resposta = "N"; 
} while (resposta === "S");

Ideal para menus e validações que precisam rodar antes de verificar.

05

O Loop for Clássico

O bom e velho contador. A única diferença do C é declarar o índice com let.

//      Início;    Condição;   Passo
for (let i = 0; i < 5; i++) {
    console.log(`Índice: ${i}`);
}

Parte 2: Controle de Fluxo

Comandando o Loop

07

Break: Abortar Missão

O break encerra o loop imediatamente. Ignora o restante das iterações.

for (let i = 0; i < 100; i++) {
    if (i === 5) {
        console.log("Achei o 5! Parando...");
        break; // Sai do loop imediatamente
    }
    console.log(i);
}

Essencial para otimizar buscas (parar assim que encontrar).

08

Continue: Pular a Vez

O continue encerra a iteração atual e força o loop a ir para a próxima.

for (let i = 0; i < 6; i++) {
    // Se for par, pula
    if (i % 2 === 0) {
        continue; // Não executa o log abaixo
    }
    console.log(i); // Só imprime ímpares: 1, 3, 5
}

Parte 3: Funções

A base da programação modular

10

1. Declaração de Função (Function Statement)

Esta é a forma mais próxima do C que vocês conhecem. A função tem nome e escopo elevado (hoisting).

C (Clássico)

// Tipo de retorno obrigatório
int somar(int a, int b) {
    return a + b;
}

JS (Declaração)

// Sem tipo, palavra 'function'
function somar(a, b) {
    return a + b;
}
11

2. Expressão de Função (Function Expression)

No JS, uma função é um valor. Podemos guardá-la dentro de uma variável.

// Guardando uma função anônima na constante 'ola'
const ola = function(nome) {
    console.log("Olá " + nome);
};

// Chamamos usando o nome da variável
ola("Maria");

Isso é vital para Callbacks (passar funções como parâmetro), muito usado em Electron para eventos de clique.

12

3. Arrow Function (Função Flecha)

Introduzida no ES6, é a sintaxe preferida no desenvolvimento moderno (React/Electron).

// Removemos a palavra 'function' e usamos '=>'
const somar = (a, b) => {
    return a + b;
};

Vantagens: Código mais limpo e comportamento diferente do this (escopo).

13

Retorno Implícito

A "Arrow Function" tem um superpoder: se a função tem apenas uma linha, podemos remover as chaves {} e o return.

// Com corpo normal
const dobro = (n) => {
    return n * 2;
};

// Com retorno implícito (Muito mais limpo!)
const dobro = n => n * 2;

*Se tiver apenas 1 parâmetro, os parênteses também são opcionais.

14

Parâmetros Especiais

Default Parameters

Valor padrão se nada for enviado.

function cor(c = "Azul") {
    console.log(c);
}
cor(); // Imprime "Azul"

Rest Parameters (...)

Aceita n argumentos como array.

function somar(...nums) {
    // nums é um array [1,2,3]
    console.log(nums.length);
}
