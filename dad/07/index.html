Parte 1: A Teoria

O que acontece por baixo dos panos?

03

O "Frankenstein" Perfeito

O Electron não é uma linguagem nova. Ele é uma casca (framework) que junta duas ferramentas gigantes que vocês já conhecem, para criar uma aplicação de PC.

Node.js (Back-end)

Cuida do "trabalho pesado".

  • Acessa arquivos do PC.
  • Conecta no Banco de Dados.
  • Abre e fecha as janelas.

Chromium (Front-end)

O "motor" do Google Chrome.

  • Desenha os botões (HTML/CSS).
  • Trata os cliques do mouse.
  • Mostra a interface para o usuário.
04

Arquitetura: Main vs Renderer

No Electron, o seu código é dividido em dois processos separados. Entender isso é a chave para não ter dor de cabeça no futuro.

Main Process

(O Gerente / O Back-end)

  • Roda em Node.js.
  • Só existe UM por aplicativo.
  • Cria as janelas.
  • Não sabe o que é um botão ou uma div.

Renderer Process

(A Janela / O Front-end)

  • Roda no Chromium.
  • Pode ter VÁRIOS (um para cada janela).
  • Renderiza o HTML/CSS/JS.
  • Não acessa o sistema operacional direto.

Pense num restaurante: O Main é a Cozinha (faz o prato pesado). O Renderer é o Salão (mostra a comida pro cliente).

Parece abstrato? Vamos sair da teoria e criar nossa primeira janela para ver isso acontecendo de verdade!

Parte 2: Mão na Massa

Chega de teoria! Vamos preparar o terreno e construir nossa primeira janela juntos.

06

1º Passo: Iniciando o Projeto

Vamos criar a base do nosso aplicativo usando o Node Package Manager (NPM).

Façam comigo no Terminal (VS Code):

  1. Crie uma pasta meu-app e abra no VS Code.
  2. Abra o terminal (Ctrl + ').
  3. Digite o comando para criar o arquivo de configuração:
npm init -y

Isso cria um arquivo package.json na sua pasta. Ele é a "identidade" do seu aplicativo.

07

2º Passo: Instalando o Electron

Agora vamos baixar a biblioteca do Electron para dentro do nosso projeto.

Façam comigo no Terminal:

Execute o comando abaixo (pode demorar alguns segundos):

npm install electron --save-dev

Note que apareceu uma pasta node_modules. NUNCA mexa nela! É aí que ficam os milhares de arquivos do Electron.

08

3º Passo: Configurando a Inicialização

Abra o arquivo package.json. Precisamos dizer a ele como o nosso app vai iniciar quando dermos o comando de play.

Façam comigo no package.json:

Encontrem a parte de "scripts" e mudem para ficar assim:

"scripts": {
  "start": "electron ."
}

Isso diz ao Node: "Quando eu rodar npm start, chame o Electron para abrir a pasta atual (o ponto significa 'aqui')".

Parte 3: O Código

Main e Renderer ganhando vida

10

4º Passo: A Interface (Renderer)

Crie um arquivo chamado index.html. Este é o rosto do nosso aplicativo (o "Salão").

Crie o index.html:

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>Meu Primeiro App</title>
</head>
<body style="background-color: #213c51; color: white; text-align: center; padding-top: 50px;">
    
    <h1>Hello World, Electron! 🚀</h1>
    <p>Esta é a minha primeira janela.</p>

</body>
</html>

Cadê a tag <script>?

Na Web clássica (Aula 03), o HTML é quem "chama" o Javascript. No Electron, a lógica se inverte! O nosso index.js (Main Process) vai nascer primeiro no sistema operacional e "puxar" este HTML para dentro da janela. Se você tentar colocar um <script src="index.js"> aqui, o app vai quebrar!

11

5º Passo: O Cérebro (Main)

Crie o arquivo index.js. Ele é o gerente (Main Process).

Crie o index.js:

const { app, BrowserWindow } = require('electron');

const criarJanela = () => {
    const janela = new BrowserWindow({
        width: 800,
        height: 600
    });

    janela.loadFile('index.html');
};

app.whenReady().then(criarJanela);
12

Dissecando: 1. A Importação

Em JavaScript nativo do navegador, não temos como criar janelas no Windows. Precisamos trazer a biblioteca do Electron para o nosso código.

require('electron')

Em C

#include <stdio.h>

O compilador C vai nos arquivos do sistema buscar as funções de input/output prontas.

Em Node.js

require('electron')

O Node vai na sua pasta oculta node_modules e carrega todo o framework para a memória.

13

Dissecando: 2. A Desestruturação

O Electron é uma caixa de ferramentas gigantesca com centenas de recursos. Se carregarmos tudo para uma variável só, o programa fica pesado e difícil de ler.

const { app, BrowserWindow } = require('electron');

A Mágica das Chaves { }

Usar chaves na criação da constante diz ao JavaScript: "Vá dentro da biblioteca Electron e puxe para mim apenas o controlador de ciclo de vida (app) e a fábrica de janelas (BrowserWindow)."

O resto da biblioteca é ignorado, economizando memória RAM.

14

Dissecando: 3. Instanciando Objetos

Ainda não estudamos Orientação a Objetos no JS, mas a lógica aqui é exatamente igual ao Java que vocês conhecem.

const janela = new BrowserWindow({ width: 800, height: 600 });

O que está acontecendo passo a passo?

  • BrowserWindow é apenas a "planta baixa" (a Classe) de uma janela genérica.
  • A palavra reservada new pede ao Sistema Operacional (Windows/Linux) para construir uma janela real e física na tela, baseada nessa planta.
  • O objeto { width: 800, height: 600 } enviado nos parênteses personaliza a construção dessa janela.
15

Dissecando: 4. O Ciclo de Vida

Um programa Desktop não liga instantaneamente. O Electron precisa de alguns milissegundos para ligar o "motor" do Chromium e se conectar aos drivers de vídeo do PC.

app.whenReady().then(criarJanela);

A Lógica Assíncrona

Leia essa linha como uma frase em português:

"Módulo app, QUANDO você estiver PRONTO (whenReady), ENTÃO (then) execute a função criarJanela."

Se o código tentasse criar a janela milissegundos antes do app estar 100% carregado, o programa travaria na hora de abrir.

16

O Grande Momento

Tudo pronto. Temos o Gerente (index.js) e o Salão (index.html).

Rodando o App

No terminal do VS Code, certifique-se de ter salvo todos os arquivos e digite:

npm start

Se tudo deu certo, uma janela independente deve se abrir na sua tela! 🎉

17

Customizando a Janela

A "fábrica" BrowserWindow aceita muitas regras para mudar o formato do seu programa. Você passa isso dentro das chaves {} na criação da janela.

Propriedade O que faz?
width / height Largura e altura iniciais em pixels (Ex: width: 800).
resizable: false Impede que o usuário mude o tamanho da janela puxando as bordas.
frame: false Remove a barra superior de título e os botões fechar/minimizar nativos.
autoHideMenuBar: true Esconde os menus "Arquivo, Editar, Ver" no topo da janela.
alwaysOnTop: true A janela fica sempre por cima de todos os outros programas do PC.
opacity: 0.8 Deixa a janela inteira transparente (vai de 0.0 até 1.0).
18

Ferramentas de Desenvolvedor (DevTools)

No navegador, usávamos o F12 para ver erros no Console. No Electron, como é um aplicativo compilado, o F12 vem desativado por padrão.

Podemos forçar a abertura do Console via código no nosso index.js:

const criarJanela = () => {
    const janela = new BrowserWindow({ width: 800, height: 600 });
    janela.loadFile('index.html');

    // Abre o painel lateral do desenvolvedor
    janela.webContents.openDevTools();
};

Isso será vital nas próximas aulas para encontrarmos os bugs no nosso Front-end!

19

Lista de Exercícios (O 1º Contato)

Estes exercícios alteram a forma como a janela se comporta. Sempre que mudar o index.js, feche o app e rode npm start novamente!

Regra de Ouro: Para o design do HTML, importem o Bootstrap ou Tailwind via CDN. Não usem CSS puro!

  1. Mini Player: Modifique a janela para abrir com exatamente 300px de largura e 150px de altura. Torne-a não redimensionável (resizable: false).
  2. Bordas Modernas: Esconda a barra de topo padrão do Windows adicionando a propriedade frame: false. Tente arrastar a janela agora. É possível fechar ela sem fechar o terminal do VSCode?
  3. Formulário de Login (Frameworks): Importe o Bootstrap ou Tailwind no seu index.html. Usando APENAS classes prontas (sem tag <style>), crie uma tela com fundo escuro, contendo um formulário de login centralizado (campos de e-mail, senha e um botão "Entrar" com estilo moderno e colorido).
  4. A Janela Fantasma: Adicione a propriedade opacity: 0.7 na configuração da janela. O que acontece com o seu aplicativo? Tente colocar alwaysOnTop: true junto.
  5. Múltiplas Janelas: Modifique o index.js. Tente criar duas variáveis janela1 e janela2, instanciando um new BrowserWindow para cada uma dentro da mesma função. Carregue o index.html nas duas. O que acontece?
