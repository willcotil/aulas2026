Parte 1: O Desafio

WebSites não são Softwares Desktop

03

Mudança de Mentalidade

Até agora, quando vocês faziam HTML, a página inteira possuía "scroll" se o conteúdo ficasse longo. Para um software Desktop (Electron), isso parece amador.

Comportamento Web

A janela inteira do navegador rola para baixo (O cabeçalho sobe e desaparece da tela).

Comportamento Desktop

A janela do Windows/App é fixa e travada. Apenas áreas específicas no meio da tela possuem barra de rolagem (ex: a lista de músicas do Spotify ou arquivos no VS Code).

O Segredo: Flexbox e 100vh

Nós precisamos forçar a tag <body> a ter exatamente o tamanho da janela e proibi-la de criar scroll. Usamos CSS para isso!

04

CSS Puro vs Frameworks

Por que não escrevemos todo o CSS na mão (tag <style>)?

Porque criar botões bonitos, cantos arredondados, sombras e grades precisas demora muito tempo. No mercado real, usamos Frameworks CSS.

CSS Puro

Dá liberdade total, mas exige que você reinvente a roda. Criar um layout de dashboard responsivo pode levar centenas de linhas de código.

A Escolha: Bootstrap 5

O padrão da indústria. Oferece um sistema de Grid perfeito para alinhar elementos e componentes prontos (Cards, Botões, Tabelas) usando classes como card, btn btn-primary, e d-flex.

Nesta disciplina, utilizaremos o Bootstrap para criar UIs rápidas e consistentes!

Parte 3: Arquitetura

O fim da bagunça no projeto

08

Por que separar as pastas?

Até à última aula, colocamos todos os arquivos (HTML, Main, Preload) soltos na mesma pasta. Isso funciona para projetos pequenos, mas num sistema real, vira um caos.

Segurança

O código do Back-end (Node) tem acesso a senhas e ao banco de dados. Ele NÃO DEVE ficar misturado fisicamente com os arquivos HTML/JS que vão para a tela do usuário.

Organização

Fica muito mais fácil encontrar um erro de botão se você souber que ele está na pasta renderer, e achar uma regra de sistema na pasta main.

09

A Árvore de Pastas Profissional

A partir de hoje, adotaremos o padrão oficial da indústria para Electron:

📁 meu-projeto/
📄 package.json
📄 dados.json // Ex: Nosso Banco de Dados
📁 src/
📁 main/ // O Back-end (Node.js)
📄 index.js // A Cozinha
📄 preload.js // O Garçom
📁 renderer/ // O Front-end (Interface/UI)
📄 index.html
📄 renderer.js // O Cliente

Parte 5: Dados Externos

Carregando arquivos JSON no Sistema

18

Separando Dados da Lógica

Em sistemas reais, não escrevemos os dados (como listas de clientes ou produtos) diretamente dentro do código JS. Nós guardamos isso em Bancos de Dados ou arquivos .json.

Um arquivo JSON nada mais é do que um arquivo de texto estruturado que o Javascript entende nativamente.

dados.json
[
  { "id": 1, "nome": "Ana", "cargo": "Dev" },
  { "id": 2, "nome": "Carlos", "cargo": "Design" }
]

Regra do JSON: Todas as chaves ("id", "nome") devem OBRIGATORIAMENTE estar entre aspas duplas!

19

Lendo o JSON e Criando a Ponte

O Node.js lê o arquivo com require(). Precisamos do Preload para expor essa função ao Front-end via invoke.

src/main/index.js
// Lê o JSON da raiz do projeto
const db = require('../../dados.json');
// Atende o telefone do front
ipcMain.handle('pedir-usuarios', () => {
return db;
});
src/main/preload.js
const { contextBridge, ipcRenderer } = require('electron');
contextBridge.exposeInMainWorld('api', {
// Cria o atalho para o Front-end
pedirUsuarios: () => {
return ipcRenderer.invoke('pedir-usuarios');
}
});
20

Renderizando na Interface

Agora o Front-end liga para o Node, pega o JSON e usa o sistema de Grid e Cards do Bootstrap para desenhar os itens na tela.

src/renderer/renderer.js
const carregarTela = async () => {
    // 1. Liga para o Node e ESPERA o JSON chegar
    const listaUsuarios = await window.api.pedirUsuarios();
    
    // 2. Mapeia os dados para gerar colunas (col) com cards do Bootstrap
    const html = listaUsuarios.map(user => `
        <div class="col">
            <div class="card shadow-sm border-0 border-start border-4 border-warning h-100">
                <div class="card-body">
                    <h5 class="card-title">${user.nome}</h5>
                    <p class="card-text text-muted">${user.cargo}</p>
                </div>
            </div>
        </div>
    `).join("");

    // 3. Joga na tela dentro de um contêiner de linha (row)
    document.querySelector("#conteudo").innerHTML = `<div class="row row-cols-1 row-cols-md-3 g-4">${html}</div>`;
};

// 4. Executa a função assim que o script carregar!
carregarTela();

Exercícios: Layout e Conexão

Juntando Estrutura de Pastas, Layout Desktop, IPC e Leitura de JSON.

  1. O Esqueleto (Bootstrap): Dentro de src/renderer/, crie o index.html. Use a técnica do Live Coding para criar uma Sidebar escura à esquerda e uma área Main clara à direita (Lembre-se do container-fluid e vh-100).
  2. A Ponte IPC: No Main (index.js), use require('../../produtos.json') e crie um ipcMain.handle('pedir-produtos'). No preload.js, crie e exponha a função pedirProdutos: () => ipcRenderer.invoke('pedir-produtos').
  3. O Grid de Cards: No renderer.js, crie a função assíncrona para buscar os dados. Na área Main do HTML, crie uma div id="catalogo". Use o map() para gerar uma coluna col contendo um componente card do Bootstrap para cada produto.

Exercícios: Dinamismo e Lógica

Tornando o Dashboard inteligente.

  1. Alerta Visual de Estoque: Modifique o seu map(). Se o estoque do produto for 0, o Card deve ter a classe border-danger e exibir um badge vermelho (<span class="badge bg-danger">ESGOTADO</span>). Caso contrário, mostre a quantidade na cor normal.
  2. Painel de Total (Topbar): Crie uma Topbar branca na sua tela. Use o método forEach() nos dados recebidos do JSON para somar: (Preço * Estoque) de todos os produtos. Exiba lá em cima: "Valor Total em Estoque: R$ X".
  3. Desafio IA (Filtro por Categoria): Adicione botões na sua Sidebar com os nomes das categorias ("Hardware", "Móveis", etc). Peça à IA: "Como crio um sistema de filtros no Front-end onde, ao clicar num botão de categoria, o meu array original de produtos vindo do IPC sofra um 'filter()' e atualize a tela mostrando apenas os produtos daquela categoria?".
