Parte 1: O Problema

O limite do Send e On

03

Relembrando a Aula Passada

Aprendemos a enviar comandos do HTML (Renderer) para o Node (Main) usando ipcRenderer.send() e ipcMain.on().

A Analogia da Carta

Usar o send() é como jogar uma carta nos Correios. Você envia a ordem (ex: "Feche o aplicativo", "Salve este arquivo") e vai cuidar da sua vida. Você não fica esperando o carteiro voltar com uma resposta.

Isso é ótimo para comandos de ação. Mas e se o Front-end precisar fazer uma pergunta?

04

A Necessidade de uma Resposta

Imagine que o seu botão HTML pede para o Node.js ler um arquivo de texto no disco rígido ou consultar um Banco de Dados.

O Jeito Antigo (Difícil)

Você enviava a pergunta com send por um canal. O Main processava. Depois o Main usava outro send para enviar a resposta por um segundo canal de volta pro Front.

Resultado: Código espaguete e difícil de ler.

O Jeito Moderno (Invoke)

Você faz uma "ligação telefônica". Você pede o dado, pausa o seu código e fica na linha esperando. Quando o Main retorna, o seu código continua com o dado em mãos.

Resultado: Código linear, limpo e direto.

Parte 2: O Telefone

Invoke, Handle e a arte de Esperar

06

A Nova Dupla Dinâmica

Esqueça o send() e o on() quando precisar de dados de volta. Apresento-lhes as novas ferramentas IPC:

ipcRenderer.invoke()

Usado no Front-end (via Preload).

Significa "Invocar". Ele chama o Back-end e cria uma Promessa (Promise) de que a resposta vai chegar.

ipcMain.handle()

Usado no Back-end (Node).

Significa "Lidar com". Ele ouve a invocação, processa o que foi pedido, e usa a palavra return para devolver a resposta pela mesma linha.

07

Passo 1: O Garçom (Preload.js)

Nosso garçom agora anota o pedido, leva na cozinha e espera o prato ficar pronto para levar de volta à mesa.

preload.js
1const { contextBridge, ipcRenderer } = require('electron');
2
3contextBridge.exposeInMainWorld('api', {
4
5 // Usa return e invoke para prometer a resposta ao HTML
6 pedirInformacao: (pergunta) => {
7 return ipcRenderer.invoke('canal-info', pergunta);
8 }
9
10});

Como invoke retorna uma Promise, repassamos esse retorno diretamente para a API do navegador.

08

Passo 2: O Cliente Esperando (HTML/JS)

Se o Garçom traz uma "Promessa", nós precisamos esperar (await) ela se cumprir antes de exibir na tela.

Para usar await, a nossa função de clique precisa ser marcada como async.

index.html (Renderer)
1const botao = document.querySelector("#btnPerguntar");
2
3botao.addEventListener("click", async () => {
4
5 // O código PAUSA nesta linha até o Node responder
6 const resposta = await window.api.pedirInformacao("Qual o sistema?");
7
8 // Só exibe o alert DEPOIS que a resposta chegar
9 alert(resposta);
10});
09

Passo 3: A Cozinha Respondendo (Main.js)

O Node usa o handle. Ele recebe o pedido, processa (pode acessar arquivos, banco, etc.) e retorna o valor pela mesma linha telefônica.

index.js (Main Process)
1const { app, BrowserWindow, ipcMain } = require('electron');
2const os = require('os'); // Módulo do Node para infos do PC
3
4app.whenReady().then(() => {
5 criarJanela();
6
7 // Handle escuta invocações no canal 'canal-info'
8 ipcMain.handle('canal-info', (evento, pergunta) => {
9 console.log("O HTML perguntou:", pergunta);
10
11 // Processa o pedido (Ex: Pega o nome do sistema operacional)
12 const sistema = os.platform();
13
14 // O return é a "mágica" que viaja de volta pro HTML
15 return `O sistema rodando é: ${sistema}`;
16 });
17});

Parte 3: Na Prática

O fluxo de ida e volta

11

O Ciclo Invoke/Handle em Ação

Renderer (HTML)

const r = await api.pedir()

Console vazio...
"Qual a versão?"

Main Process (Node)

ipcMain.handle(...)

Aguardando invoke...
  1. Front-end usa o await, o que pausa o seu script HTML (Loading).
  2. O Main recebe o pedido via handle e pode fazer operações demoradas (Ler banco de dados).
  3. O Main dá um return. O pacote volta pela mesma linha. O await do HTML é destravado com a resposta!

Parte 4: Live Coding

Prática Guiada: O Buscador do Sistema

O que vamos construir?

Um aplicativo onde o Front-end (HTML) faz uma "ligação" para o Back-end (Node.js) perguntando o Nome do Usuário logado no computador e a quantidade de Memória RAM instalada. O Node.js fará essa leitura direto do sistema operacional e devolverá os dados para exibirmos na interface gráfica!

Antes de programar, precisamos de iniciar o projeto Node.js e instalar o Electron. Abram o terminal do VS Code (Ctrl + ') numa pasta vazia.

Crie o package.json (Identidade do app):

2. No package.json

Abra o ficheiro gerado e altere o bloco "scripts" para ensinar o Node a iniciar a nossa janela:

Passo 2: O Garçom (A Ponte)

O HTML não grita direto para o Node. Aqui usamos a palavra mágica invoke, que pede algo e espera uma promessa de resposta.

preload.js
const { contextBridge, ipcRenderer } = require('electron');

contextBridge.exposeInMainWorld('api', {
    
    // Criamos um "telefone" chamado buscarInfoPC
    buscarInfoPC: () => {
        // O invoke dispara a mensagem pelo canal 'get-sys-info'
        // e PROMETERÁ devolver o que o Main responder!
        return ipcRenderer.invoke('get-sys-info');
    }

});
16

Passo 3: O Back-end (A Cozinha)

O garçom grita 'get-sys-info'. A cozinha usa o handle para ouvir, prepara o dado e usa o return para devolver a bandeja.

index.js
const { app, BrowserWindow, ipcMain } = require('electron');
const path = require('path');
const os = require('os'); // Módulo do Node para infos do PC

function criarJanela() {
    const janela = new BrowserWindow({
        width: 600, height: 400,
        webPreferences: { preload: path.join(__dirname, 'preload.js') }
    });
    janela.loadFile('index.html');
}

app.whenReady().then(() => {
    criarJanela();

    // A COZINHA ATENDE O TELEFONE AQUI
    ipcMain.handle('get-sys-info', () => {
        console.log("O Front-end pediu os dados. Calculando...");
        const nomeUsuario = os.userInfo().username;
        const memoriaTotalMB = Math.round(os.totalmem() / 1024 / 1024);

        // O SEGREDO DO BIDIRECIONAL: O return!
        return `Usuário: ${nomeUsuario} | Memória RAM: ${memoriaTotalMB} MB`;
    });
});
17

Passo 4: O Cliente Esperando

Como o Node.js pode demorar, a nossa função de clique precisa ser async e mandamos o código esperar a resposta com await.

renderer.js
const btnBuscar = document.getElementById('btnBuscar');
const pResultado = document.getElementById('resultado');

// O 'async' avisa: "Esta função vai ter pausas"
btnBuscar.addEventListener('click', async () => {
    
    pResultado.innerText = "Consultando o Sistema... ⏳";
    
    // O 'await' pausa a execução EXATAMENTE NESTA LINHA.
    const respostaDoNode = await window.api.buscarInfoPC();

    // A resposta chegou! Vamos mostrar na tela.
    pResultado.innerText = respostaDoNode;
});

Lista de Exercícios (Bidirecional)

Configure o Preload com invoke e resolva os desafios abaixo construindo a ponte de ida e volta.

  1. Ping-Pong: Crie um botão "Ping". O HTML faz o invoke mandando "Ping". O Main recebe, processa e dá um return "Pong". O HTML deve exibir a resposta usando um alert().
  2. Calculadora Node: O HTML possui dois inputs numéricos e um botão "Somar no Back-end". Envie os dois números (em um objeto {n1: x, n2: y}) pelo invoke. O Main soma e retorna o total para o HTML exibir num parágrafo.
  3. Info do Sistema: Crie um botão "Meu PC". O Main deve importar o require('os') e, no handle, retornar um objeto com a Memória Livre (os.freemem()) e a versão (os.release()). Exiba isso formatado no HTML.
  4. Gerador de Senha Forte: O HTML envia o tamanho desejado (ex: 12). O Main gera uma senha aleatória segura misturando letras e números e retorna a string gerada para o Front-end mostrar na tela.
  5. Desafio da Espera Fake: No HTML, ao clicar no botão "Baixar Dados", troque o texto do botão para "Baixando...". No Main, crie um setTimeout de 3 segundos antes de dar o return "Dados Concluídos!" (você precisará retornar uma Promise). Quando a resposta chegar no HTML, mude o botão para "Concluído!".
  6. Desafio IA (Árvore de Arquivos VS Code): Vá muito além da lista simples! Peça à IA: "Como ler de forma recursiva pastas e arquivos usando o módulo 'fs' no Node.js, retornar essa estrutura hierárquica via IPC (handle), e criar no Front-end uma árvore de arquivos expansível (estilo VS Code) muito bem estilizada com HTML e CSS?". O desafio é implementar a lógica no Node e criar a interface. Atenção: adicione um toque pessoal (um tema de cores único, ícones específicos para o tipo da extensão do arquivo ou exiba o tamanho/peso do arquivo ao lado do nome) para diferenciar o seu explorador do projeto dos seus colegas!
