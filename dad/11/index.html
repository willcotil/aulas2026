A Mágica das 12 Colunas

O Bootstrap divide a tela em 12 colunas invisíveis. Quando você cria uma div, você diz quantas colunas ela deve ocupar.

Como funciona?

Você cria um container de linha <div class="row"> e coloca colunas <div class="col-..."> dentro.

  • col-12: Ocupa a tela inteira (12/12).
  • col-6: Ocupa metade da tela (6/12).
  • col-4: Ocupa um terço da tela (4/12).

Breakpoints

Podemos dizer ao Bootstrap para mudar o tamanho dependendo da largura da tela usando prefixos:

  • sm (Small / Celular Deitado)
  • md (Medium / Tablet)
  • lg (Large / Desktop)

Parte 2: Inputs & Formulários

Lendo dados do usuário do jeito profissional

06

A Armadilha do <form>

Na Web antiga, clicar em "Salvar" num formulário fazia o navegador pegar os dados, enviar para o servidor e recarregar a página inteira.

O Problema no Electron e React

Se o seu HTML recarregar, todo o Javascript do Front-end (variáveis, listas, estado do app) será apagado da memória RAM. A tela vai piscar e voltar ao zero!

Nós PRECISAMOS interceptar o envio e impedir que a janela recarregue.

07

O Salvador: preventDefault()

A primeira regra de um formulário moderno: interceptar o evento submit e usar o comando preventDefault() para travar o comportamento destrutivo do navegador.

renderer.js
const form = document.querySelector("#formProduto");

// Escutamos o evento de envio (submit), e não o clique do botão
form.addEventListener("submit", (e) => {
    // 1. IMPEDE O RECARREGAMENTO DA PÁGINA (CRÍTICO!)
    e.preventDefault();
    
    // Agora a tela está travada e segura. 
    // Próximo passo: pegar os dados que o usuário digitou!
});
08

Coletando Dados: O Antigo vs O Moderno

Até agora, usávamos document.querySelector("#id").value. Imagine um formulário de cadastro de cliente com 20 campos (Nome, CPF, Rua, Bairro, etc). O seu código viraria um pesadelo espaguete.

O Jeito Amador

const nome = document.querySelector("#nome").value;
const cpf = document.querySelector("#cpf").value;
const cep = document.querySelector("#cep").value;
// ... (mais 17 linhas iguais) ...
const obj = { nome: nome, cpf: cpf, cep: cep ... }

O Jeito Profissional

Deixamos a API nativa do navegador, chamada FormData, varrer o formulário e agrupar tudo automaticamente em uma única linha de código!

09

O Segredo do FormData: O Atributo name

Para o `FormData` funcionar, a única coisa que o HTML exige é que todos os seus inputs tenham o atributo `name`. O `id` não importa para ele.

index.html
<form id="formCadastro">
    <!-- Texto e Números -->
    <input type="text" name="produto" placeholder="Nome">
    <input type="number" name="quantidade" placeholder="Qtd">
    
    <!-- Rádio (Opção Única): Mesma 'name', 'value' diferentes -->
    <input type="radio" name="categoria" value="Hardware"> Hardware
    <input type="radio" name="categoria" value="Software"> Software
    
    <!-- Slider e Checkbox -->
    <input type="range" name="desconto" min="0" max="100">
    <input type="checkbox" name="promocao"> Em Promoção?
    
    <button type="submit">Salvar</button>
</form>
10

A Extração Mágica

No Javascript, passamos o elemento do formulário para o new FormData(), e depois usamos Object.fromEntries() para transformar os dados num JSON perfeito!

renderer.js
const form = document.querySelector("#formCadastro");

form.addEventListener("submit", (e) => {
    e.preventDefault(); // 1. Trava o recarregamento
    
    // 2. O FormData "suga" os valores baseados no atributo 'name'
    const extrator = new FormData(form);
    const dadosProntos = Object.fromEntries(extrator);
    
    console.log(dadosProntos);
    /* Saída: 
       { produto: "Teclado", quantidade: "2", categoria: "Hardware", 
         desconto: "15", promocao: "on" } 
    */
    
    form.reset(); // Limpa todos os campos da tela
});

Pesquisa Instantânea

Diferente de um formulário que precisa do botão "Salvar", uma barra de pesquisa moderna filtra os dados a cada letra que o usuário digita.

Para isso, não usamos o evento click ou submit. Usamos o evento input num campo de texto isolado.

O Gatilho

const barraBusca = document.querySelector("#busca");

barraBusca.addEventListener("input", () => {
    const texto = barraBusca.value.toLowerCase();
    // A cada letra digitada, o código roda!
});

A Lógica

Dentro desse evento, usamos o filter() no nosso array original de dados para encontrar itens cujo nome inclua o texto digitado (includes()).

Passo 3: A Lógica da Busca

Agora adicionamos o "escutador" na barra de busca que fará o filtro e desenhará o catálogo novamente (usando a lista filtrada)!

src/renderer/renderer.js (Parte 2)
// ... código anterior ...

const barraBusca = document.querySelector("#inputBusca");

barraBusca.addEventListener("input", () => {
    const textoDigitado = barraBusca.value.toLowerCase();
    
    // Filtra a lista global baseada no que foi digitado
    const listaFiltrada = listaProdutos.filter(prod => {
        return prod.nome.toLowerCase().includes(textoDigitado);

    desenharCards(listaFiltrada); 
});

Passo 5: A Lógica do Cadastro

No renderer.js, interceptamos o evento submit, usamos a magia do FormData para extrair os dados, e injetamos no Array Global!

src/renderer/renderer.js (Parte 3)
const form = document.querySelector("#formProduto");

form.addEventListener("submit", (e) => {
    e.preventDefault(); // Trava o recarregamento da página!
    
    const extrator = new FormData(form);
    const novoProduto = Object.fromEntries(extrator);
    
    // Adiciona na lista global e redesenha a tela!
    listaProdutos.push(novoProduto);
    desenharCards(listaProdutos);
    
    form.reset(); // Limpa os inputs
});

Exercícios: Responsividade & Busca

Use o projeto da Aula 10 (Dashboard de Inventário) como base.

  1. Grid Responsivo: No seu renderer.js (Aula 10), o Grid dos cards estava fixo em 3 colunas (row-cols-md-3). Altere a string HTML do container no JavaScript para que fique: 1 coluna no celular (col-12), 2 colunas em tablets (col-md-6) e 4 colunas em telas grandes (col-lg-3). Redimensione a janela do seu app para testar.
  2. Separando Responsabilidades: Refatore seu renderer.js. Crie uma função buscarDados() que faz o invoke e salva na variável global, e uma função desenharTela(lista) que recebe um array e faz o map().
  3. A Busca Dinâmica: Implemente a barra de busca (vista no Live Coding). Ao digitar qualquer letra na Topbar, o Dashboard deve atualizar instantaneamente mostrando apenas os produtos cujo nome ou categoria correspondam ao texto digitado.

Exercícios: Aprimorando o Cadastro

  1. Evoluindo o Formulário: O formulário do Live Coding está muito simples. Adicione mais dois campos HTML nele: Um <select name="categoria"> com opções (Hardware, Periféricos, Móveis) e um input de quantidade. Como usamos o FormData, o seu JS vai capturar esses campos novos automaticamente sem você mudar 1 linha de JS! Mude a função de desenhar os cards para exibir essas novas informações na tela.
