Lendo dados do usuário do jeito profissional

02

A Armadilha do <form>

Na Web antiga, clicar em "Salvar" num formulário fazia o navegador pegar os dados, enviar para o servidor e recarregar a página inteira.

O Problema no Electron e React

Se o seu HTML recarregar, todo o Javascript do Front-end (variáveis globais, listas, estado do app) será apagado da memória RAM. A tela vai piscar e voltar ao zero!

Nós PRECISAMOS interceptar o envio e impedir que a janela recarregue.

03

O Salvador: preventDefault()

A primeira regra de um formulário moderno: interceptar o evento submit e usar o comando preventDefault() para travar o comportamento destrutivo do navegador.

renderer.js
const form = document.querySelector("#formProduto");

// Escutamos o evento de envio (submit), e não o clique do botão
form.addEventListener("submit", (e) => {
    // 1. IMPEDE O RECARREGAMENTO DA PÁGINA (CRÍTICO!)
    e.preventDefault();
    
    // Agora a tela está travada e segura. 
    // Próximo passo: pegar os dados que o usuário digitou!
});
04

Coletando Dados: O Antigo vs O Moderno

Até agora, usávamos document.querySelector("#id").value. Imagine um formulário de cadastro de cliente com 20 campos (Nome, CPF, Rua, Bairro, etc). O seu código viraria um pesadelo espaguete.

O Jeito Amador

const nome = document.querySelector("#nome").value;
const cpf = document.querySelector("#cpf").value;
const cep = document.querySelector("#cep").value;
// ... (mais 17 linhas iguais) ...
const obj = { nome: nome, cpf: cpf, cep: cep ... }

O Jeito Profissional

Deixamos a API nativa do navegador, chamada FormData, varrer o formulário e agrupar tudo automaticamente em uma única linha de código!

05

O Segredo do FormData: O Atributo name

Para o `FormData` funcionar, a única coisa que o HTML exige é que todos os seus inputs tenham o atributo `name`. O `id` não importa para ele.

index.html
<form id="formCadastro">
    <!-- Texto e Números -->
    <input type="text" name="produto" placeholder="Nome">
    <input type="number" name="quantidade" placeholder="Qtd">
    
    <!-- Rádio (Opção Única): Mesma 'name', 'value' diferentes -->
    <input type="radio" name="categoria" value="Hardware"> Hardware
    <input type="radio" name="categoria" value="Software"> Software
    
    <!-- Slider e Checkbox -->
    <input type="range" name="desconto" min="0" max="100">
    <input type="checkbox" name="promocao"> Em Promoção?
    
    <button type="submit">Salvar</button>
</form>
06

A Extração Mágica

No Javascript, passamos o elemento do formulário para o new FormData(), e depois usamos Object.fromEntries() para transformar os dados num JSON perfeito!

renderer.js
const form = document.querySelector("#formCadastro");

form.addEventListener("submit", (e) => {
    e.preventDefault(); // 1. Trava o recarregamento
    
    // 2. O FormData "suga" os valores baseados no atributo 'name'
    const extrator = new FormData(form);
    const dadosProntos = Object.fromEntries(extrator);
    
    console.log(dadosProntos);
    /* Saída: 
       { produto: "Teclado", quantidade: "2", categoria: "Hardware", 
         desconto: "15", promocao: "on" } 
    */
    
    form.reset(); // Limpa todos os campos da tela
});

Parte 2: Busca Dinâmica

Filtros em tempo real (Estilo Google)

08

Pesquisa Instantânea

Diferente de um formulário que precisa do botão "Salvar", uma barra de pesquisa moderna filtra os dados a cada letra que o usuário digita.

Para isso, não usamos o evento click ou submit. Usamos o evento input num campo de texto isolado.

O Gatilho

const barraBusca = document.querySelector("#busca");

barraBusca.addEventListener("input", () => {
    const texto = barraBusca.value.toLowerCase();
    // A cada letra digitada, o código roda!
});

A Lógica

Dentro desse evento, usamos o filter() no nosso array original de dados para encontrar itens cujo nome inclua o texto digitado (includes()).

Passo 1: Refatorando o `renderer.js`

Substitua todo o código antigo do seu `renderer.js` por esta versão modular. Separamos a função que desenha a tela (agora recebe um parâmetro) da função que inicializa o app.

renderer.js (Código Completo Base)
let listaProdutos = []; // A variável agora é GLOBAL para que a busca possa ler!

const desenharTela = (listaParaDesenhar) => {
    const htmlCards = listaParaDesenhar.map(produto => {
        const bordaEstoque = produto.estoque > 0 ? 'border-warning' : 'border-danger';
        const badgeEstoque = produto.estoque > 0 ? '' : '<span class="badge bg-danger">ESGOTADO</span>'; 
        
        return `
        <div class="col">
            <div class="card shadow-sm border-0 border-start border-4 ${bordaEstoque} h-100">
                <div class="card-body">
                    <h5 class="card-title">${produto.nome} ${badgeEstoque}</h5>
                    <p class="card-text text-muted mb-1">Preço: ${produto.preco.toFixed(2)}</p>
                    <p class="card-text text-muted mb-0">Estoque: ${produto.estoque}</p>
                </div>
            </div>
        </div>`
    }).join("");

    document.querySelector("#conteudo").innerHTML = `<div class="row row-cols-1 row-cols-md-3 g-4">${htmlCards}</div>`;

    // Re-calcula o total baseado na lista que foi desenhada!
    let total = 0;
    listaParaDesenhar.forEach(produto => {
        total += (produto.estoque * produto.preco);
    });
    document.querySelector("#total").innerText = `Total: ${total.toFixed(2)}`;
};

const iniciarApp = async () => {
    listaProdutos = await window.api.pedirProdutos(); // Busca os dados do Node
    desenharTela(listaProdutos); // Manda desenhar a lista completa
};

iniciarApp(); // Dá a partida no programa

Passo 3: A Lógica da Busca (JS)

No final do seu renderer.js (abaixo de onde você chamou `iniciarApp();`), cole o código que vai escutar a digitação e filtrar a tela.

renderer.js (Cole no final do arquivo)
// ---------------------------------------------
// LÓGICA DE PESQUISA (Adicionado no Passo 3)
const barraBusca = document.querySelector("#inputBusca");

barraBusca.addEventListener("input", () => {
    const textoDigitado = barraBusca.value.toLowerCase();
    
    // Filtramos a listaGlobal usando includes()
    const listaFiltrada = listaProdutos.filter(produto => {
        return produto.nome.toLowerCase().includes(textoDigitado);
    });
    
    // Mandamos a tela se redesenhar com a lista cortada!
    desenharTela(listaFiltrada); 
});

Passo 5: A Magia do FormData (JS)

Para finalizar, no final do seu renderer.js, cole a lógica que intercepta o envio do form, extrai os dados, salva e redesenha!

renderer.js (Cole no final do arquivo)
// ---------------------------------------------
// LÓGICA DE CADASTRO (Adicionado no Passo 5)
const form = document.querySelector("#formProduto");

form.addEventListener("submit", (e) => {
    e.preventDefault(); // Trava o recarregamento da página!
    
    // Extrai tudo de uma vez usando os 'names' do HTML
    const extrator = new FormData(form);
    const novoProduto = Object.fromEntries(extrator);
    
    // Converte texto pra número pra não bugar cálculos matemáticos
    novoProduto.preco = Number(novoProduto.preco);
    novoProduto.estoque = Number(novoProduto.estoque);

    // Adiciona no Array global
    listaProdutos.push(novoProduto);
    
    // Manda desenhar de novo a tela com o produto novo
    desenharTela(listaProdutos);
    
    form.reset(); // Limpa os inputs
});

Desafios Finais do Dashboard

Você já tem um software que pesquisa e adiciona itens. Agora melhore a usabilidade!

  1. Formatação de Moeda: O JavaScript possui uma ferramenta nativa chamada Intl.NumberFormat para moedas. Pesquise como utilizá-la e altere o seu `renderer.js` para que os preços dos produtos apareçam formatados como Moeda Brasileira (ex: R$ 1.200,00), em vez de usar apenas o toFixed(2).
  2. Validação de Preço: No evento submit do seu formulário, adicione uma trava lógica (if). Se o preço digitado for menor ou igual a ZERO, exiba um alert() com mensagem de erro e use o return para abortar a função antes do .push() acontecer.
  3. Filtro de Categoria: Adicione um input name="categoria" no seu form do HTML. Como estamos usando FormData, você não precisará alterar nada no seu JS de envio para que esse dado seja capturado! Apenas exiba a categoria no HTML do Card.
  4. Desafio IA (Exclusão): Adicione um botão "X" vermelho em cada Card gerado. Peça ajuda à IA com o prompt: "Tenho um array global renderizado na tela. Adicionei um botão de deletar em cada item. Como faço no JS para que, ao clicar nesse botão, eu identifique qual produto foi clicado, remova ele do Array Global e re-chame a função que desenha a tela?".
