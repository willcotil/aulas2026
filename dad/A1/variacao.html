📋 Diretrizes da Avaliação

Esta seção detalha o escopo do desafio proposto aos alunos e as regras de engajamento para a construção do painel de gestão do catálogo de peças cibernéticas. O entendimento claro destas regras é vital para a avaliação.

O que o programa deve fazer

  • 1. Array de objetos inicial com peças de exemplo (ex: {nome: "Olho", preco: 2500, altoRisco: false}).
  • 2. Formulário HTML (Inputs e Botão) para registrar um novo implante no catálogo.
  • 3. Listagem de implantes na tela. Itens com altoRisco: true devem ter destaque visual (⚠️).
  • 4. Botão "Mostrar Apenas Seguros" que oculta implantes de alto risco via filtro JS.
  • 5. Painel exibindo o "Custo Total do Catálogo" calculado dinamicamente.

Conjunto A

🟦
Pergunta: Por que o uso de Number(implante.preco) é necessário ao trabalhar com inputs do HTML?

Gabarito: Tudo o que vem de um input HTML via .value chega ao Javascript como String. Se não convertermos usando Number(), o operador + irá concatenar os textos (ex: "15" + "5" = "155") em vez de somar matematicamente.
Pergunta: O que aconteceria se não usássemos o .join("") após o .map() no innerHTML?

Gabarito: O map retorna um Array. O HTML imprimiria as vírgulas que separam os itens do array na tela. O .join("") junta todas as strings do array em uma única string, removendo as vírgulas indesejadas.
Pergunta: Por que faz sentido usar const para elementos da tela (ulLista) e let para o array (catalogo)?

Gabarito: const é usado pois a referência na memória daquele elemento HTML não muda na execução; tentar reatribuir geraria erro. O array usa let para indicar semanticamente que o catálogo sofrerá grandes alterações de estado ao longo do tempo.

Conjunto B

🟩
Pergunta: Explique a verificação if (!nome || !preco) e o uso do return alert.

Gabarito: O ! verifica valores falsy (vazio). O || é OU. Testa: "Se nome for vazio OU preço for vazio". O return aborta a função imediatamente, impedindo a criação de um objeto com dados em branco.
Pergunta: O .filter() altera ou deleta dados do array original catalogo?

Gabarito: Não. O filter avalia a condição e gera um novo array contendo apenas cópias dos itens que passaram no teste, preservando o array original intacto.
Pergunta: Traduza classeCSS = altoRisco ? "risco" : "seguro"; para um bloco if/else comum.

Gabarito:
let classeCSS;
if (implante.altoRisco == true) {
    classeCSS = "risco";
} else {
    classeCSS = "seguro";
}

Conjunto C

🟧
Pergunta: No addEventListener, por que não colocamos parênteses na função chamada no segundo argumento?

Gabarito: O listener precisa da "receita" da função (callback) para executá-la apenas no futuro (no clique). Colocar parênteses executaria a função imediatamente durante o carregamento da página.
Pergunta: Quais as diferenças visuais da Arrow Function em comparação com uma função C clássica?

Gabarito: Ausência da palavra function, não exige definição de tipo (int/void), e utiliza o sinal de flecha => após os parâmetros.
Pergunta: Faz sentido usar .map() apenas para calcular o somatório de preços?

Gabarito: Não. O .map() serve para transformar dados gerando um novo array. O .forEach() serve apenas para iterar e executar ações colaterais (como somar valores numa variável externa). Usar map apenas para somar é desperdício de memória.
