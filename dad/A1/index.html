Avaliação Prática 1

Gerenciador de Inventário Espacial

Vocês foram contratados para desenvolver o painel de controle do inventário de uma nave espacial. Usem HTML, CSS e Javascript para cumprir a missão.

Requisitos do Sistema

  1. Ter um array de objetos inicial com alguns itens de exemplo.
    Ex: {nome: "Cilindro de Oxigênio", quantidade: 5, critico: true}.
  2. Ter um formulário no HTML (Inputs e Botão) para adicionar um novo item dinamicamente ao inventário.
  3. Mostrar a lista de itens na tela. Se o item for critico: true, ele deve aparecer com destaque visual (ex: cor vermelha ou um emoji de alerta ⚠️).
  4. Ter um botão "Filtrar Críticos" que, ao ser clicado, mostra na tela apenas os itens críticos.
  5. Ter um painel (indicador) que mostre o total de itens armazenados na nave (a soma das quantidades de todos os itens).

Regras de Engajamento

Pesquisa Livre

Podem usar o Google, YouTube, ChatGPT ou consultar os colegas durante o desenvolvimento em casa.

Uso Obrigatório

É obrigatório o uso de Arrow Functions, os métodos map e filter, e a manipulação do DOM via Javascript.

Entrega Prévia do Código

Vocês devem enviar o código que desenvolveram em casa para o professor. A entrega deve ser feita no Google Classroom até 1 dia antes da avaliação. A entrega do projeto de vocês é um pré-requisito obrigatório para participar da sabatina.

Atenção à Avaliação (Defesa de Código)

O objetivo de fazerem o projeto em casa é estudarem o problema e a lógica. No dia da prova, o código avaliado NÃO SERÁ O DE VOCÊS, será o código do professor.

A avaliação será uma sabatina oral de 5 perguntas sobre o funcionamento, estrutura e lógica do código oficial projetado no telão.
