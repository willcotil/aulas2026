Parte 1: O Bloqueio

Segurança em primeiro lugar

03

O Abismo de Segurança

Na Aula 07 vimos que o Main Process é o Node.js (tem poderes totais) e o Renderer é o Chromium (apenas a tela).

Tente imaginar:

Se o código HTML/JS do seu botão pudesse deletar arquivos do Disco Local C: diretamente, qualquer falha no site (ou script injetado) destruiria o computador do usuário.

Por padrão de segurança, o Electron CORTA a comunicação direta entre o Front-end e o Node.js.

04

A Solução: IPC

Como um botão HTML (Renderer) avisa o Node.js (Main) para fechar o programa ou salvar um arquivo de texto?

Através do IPC (Inter-Process Communication). É um sistema de rádio-transmissor interno do Electron.

HTML / JS

Clica no Botão "Salvar"

Rádio (IPC)

Manda a mensagem "SALVAR_ARQUIVO"

Main Process

Ouve a mensagem e salva no Disco Rígido

05

O Arquivo preload.js

Para usar o rádio IPC com segurança, o Electron nos obriga a criar um terceiro arquivo: o Preload.

A Analogia do Restaurante:

Renderer (Salão)

O Cliente que olha o cardápio e faz o pedido (Interface gráfica). Ele não pode entrar na cozinha.

Preload (Garçom)

O único autorizado a falar com o Cliente e a entrar na Cozinha. Leva os pedidos de um lado para o outro de forma segura.

Main (Cozinha)

O Chef que tem as panelas e facas (Node.js). Prepara o prato pesado e interage com o sistema operacional.

Parte 2: Configuração

Ajustando o Main e criando o Garçom

07

Passo 1: Contratando o Garçom

No nosso index.js (Main), precisamos dizer à janela para carregar o script preload.js antes do HTML aparecer.

index.js (Main Process)
1const { app, BrowserWindow } = require('electron');
2const path = require('path'); // Módulo do Node para caminhos
3
4const criarJanela = () => {
5 const janela = new BrowserWindow({
6 width: 800, height: 600,
7 webPreferences: {
8 // "Precarrega" este script antes de tudo
9 preload: path.join(__dirname, 'preload.js')
10 }
11 });
12
13 janela.loadFile('index.html');
14};

path.join(__dirname, 'preload.js') garante que ele ache o arquivo independente de ser Windows ou Mac.

08

Passo 2: O Preload e a ContextBridge

Agora crie um arquivo vazio chamado preload.js.

Usaremos a ferramenta contextBridge para criar uma "Ponte Segura". Ela pega funções do lado de cá (Node) e expõe para o lado de lá (HTML/Window).

preload.js (O Garçom)
1const { contextBridge, ipcRenderer } = require('electron');
2
3// "Exponha no Mundo Principal (HTML)"
4contextBridge.exposeInMainWorld('api', {
5
6 // Eu crio um comando chamado "enviarMensagem"
7 // que o HTML vai poder usar!
8 enviarMensagem: (texto) => {
9 // Envia via Rádio IPC para o Main
10 ipcRenderer.send('canal-mensagem', texto);
11 }
12
13});

Passo 3: Usando a API no Front-end

Aquele objeto 'api' que criamos no Preload agora está injetado magicamente no nosso navegador (na variável global window).

Passo 4: Escutando no Main

O Front enviou a mensagem pelo canal 'canal-mensagem'. Mas o Main precisa estar "ouvindo" essa estação de rádio para agir.

index.js (Main Process)
1const { app, BrowserWindow, ipcMain } = require('electron');
2const path = require('path');
3
4const criarJanela = () => {
5 const janela = new BrowserWindow({
6 width: 800, height: 600,
7 webPreferences: { preload: path.join(__dirname, 'preload.js') }
8 });
9 janela.loadFile('index.html');
10};
11
12app.whenReady().then(() => {
13 criarJanela();
14
15 // Liga o rádio na estação 'canal-mensagem'
16 ipcMain.on('canal-mensagem', (evento, dados) => {
17 console.log("Mensagem recebida do HTML: ", dados);
18 });
19});
12

O Ciclo IPC em Ação

Renderer (HTML)

window.api.enviar()

"Olá Main!"

Main Process (Node)

ipcMain.on(...)

Esperando...
  1. O HTML não conhece o Node. Ele só conhece o objeto window.api.
  2. O Preload (que montou a API) pega a string e dispara o ipcRenderer.send().
  3. O Main ouve o evento via ipcMain.on() e executa o código pesado (console/arquivos).
13

A Ponte para o Padrão MVC

Tudo isso que acabamos de montar forma a base de uma das arquiteturas mais famosas do mercado: o MVC (Model-View-Controller).

V - View (Visão)

O seu HTML/CSS. Apenas exibe os botões e interage com o utilizador. É "burra", não sabe acessar bancos de dados nem salvar arquivos.

C - Controller (Controle)

O Preload e o IPC. Recebe os cliques da View, entende o que o usuário quer fazer, e roteia a ordem para o sistema central. É o maestro.

M - Model (Modelo)

O Main (Node.js). Onde o trabalho pesado e as regras de negócio reais acontecem. Consulta o banco de dados (MySQL) ou salva documentos no PC.

Manter as responsabilidades separadas é o segredo de um código limpo!

14

Exemplo Real: Botão de Sair

Se você removeu as bordas da janela na Aula 07 (frame: false), como o usuário fecha o app? Criando um botão de Sair no HTML!

1. Preload.js

contextBridge.exposeInMainWorld('api', {
  fecharApp: () => ipcRenderer.send('fechar')
});

2. index.html

botao.addEventListener('click', () => {
  window.api.fecharApp();
});

3. index.js (Main)

ipcMain.on('fechar', () => {
    app.quit(); // Comando poderoso do Node/Electron para matar o processo
});

Lista de Exercícios (O Rádio IPC)

Crie a estrutura completa (HTML, Preload e Main) para resolver os desafios abaixo enviando comandos Front -> Back.

  1. Alarme Falso: Crie um botão no HTML "Ativar Alarme". Ao clicar, envie uma mensagem IPC. O Main deve interceptar e dar um console.log("ALARM ALARM ALARM") três vezes no terminal do VSCode.
  2. Botão de Pânico (Sair): Retire as bordas da janela (frame: false). Crie um botão HTML vermelho escrito "Fechar Sistema" que usa IPC para acionar o app.quit() no Main.
  3. Janela Minimizada: Crie um botão "Minimizar" no HTML. Use IPC. No Main, você pode minimizar a janela atual usando BrowserWindow.getFocusedWindow().minimize().
  4. Redimensionador Dinâmico: Crie dois inputs no HTML (Largura e Altura) e um botão "Mudar Tamanho". Envie os dois valores via IPC. O Main deve usar janela.setSize(largura, altura).
  5. Modo Noturno Nativo: Crie um botão "Dark Mode". Envie um comando IPC. No Main, mude a cor de fundo da própria BrowserWindow para preto usando o comando janela.setBackgroundColor('#000').
