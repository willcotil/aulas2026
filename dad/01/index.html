02

Evolução da Plataforma

Até agora, vocês aprenderam o jeito "clássico". Agora vamos para o jeito "moderno".

Windows Forms (C#)

O que vocês já viram.

  • Interface: Arrastar botões (Drag & Drop).
  • Estilo: Visual padrão do Windows (difícil mudar).
  • Portabilidade: Roda apenas no Windows.
  • Ideal para: Sistemas legados, ferramentas internas rápidas.

Electron (JS/Web)

O que vamos aprender.

  • Interface: HTML & CSS (Liberdade total de design).
  • Estilo: Moderno (igual Spotify, Discord).
  • Portabilidade: Roda em Windows, Linux e Mac.
  • Ideal para: Produtos comerciais modernos.
03

Por que mudar para Electron?

Se o C# funciona, por que mudar?

  1. Mercado: A demanda por interfaces bonitas e modernas é alta. O WinForms tem cara de "anos 2000".
  2. Reaproveitamento: O que você aprende aqui (HTML/JS) serve para criar Sites e aplicativos Mobile também.
  3. Ecossistema: O Node.js tem milhões de bibliotecas prontas (NPM) que não existem no C# nativo.

"Escreva uma vez, rode em todo lugar."

05

A Base de Tudo: Node.js

Para rodar JS fora do navegador, precisamos do Node.js.

Compilador C/C#

Gera um arquivo .exe binário antes de rodar.

Runtime Node.js

Lê o arquivo de texto .js e executa na hora (JIT Compilation).

O Node.js será o nosso "motor".

06

Instalando o Node.js

Escolha o procedimento de acordo com seu Sistema Operacional:

Windows

1. Acesse nodejs.org.

2. Baixe a versão LTS (Recomendada).

3. Execute o instalador (.msi) e siga com "Next" até o fim.

4. Reinicie o computador.

Linux

Abra o terminal e execute:

sudo apt update
sudo apt install nodejs npm

Ou, para versões mais novas:

curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs

Teste Final: Abra o terminal e digite node -v

07

Como Executar um Código?

O Node não tem uma "janela" própria. Ele roda no terminal.

1. O Arquivo

Crie um arquivo de texto com a extensão .js (ex: teste.js).

Coloque o seguinte conteúdo:

console.log("Olá Mundo!");

2. A Execução

No terminal, navegue até a pasta do arquivo e digite:

node teste.js

Comparação de Sintaxe

Javascript é muito parecido com C# na estrutura básica.

C / C#

  • Tipagem Forte (int, string)
  • Compilado
  • Estrutura Rígida

Javascript

  • Tipagem Dinâmica (let, const)
  • Interpretado
  • Flexível

Ambos usam chaves {} e ponto-e-vírgula ;.

15

Sintaxe de Declaração

  • const: Para valores que não mudam (Use sempre que possível).
  • let: Para valores que vão mudar.
  • var: Não use (obsoleto).
const PI = 3.14;
let contador = 0;
contador = 1; // OK
// PI = 3.15; // ERRO!

Comparação Estrita

Cuidado com o ==!

Funções: Arrow Functions

A sintaxe moderna e curta para funções.

Função Clássica

function somar(a, b) {
    return a + b;
}

Arrow Function

const somar = (a, b) => {
    return a + b;
}
23

Fixação Rápida

Transforme esta função clássica em uma Arrow Function:

function subtrair(a, b) {
    return a - b;
}

Dica: Use const para guardar a função.

24

Objetos (JSON) vs Structs

Objetos em JS são mais flexíveis e fáceis de escrever que Structs em C.

C (Struct)

struct Aluno {
    char nome[50];
    float nota;
};

struct Aluno a1;
strcpy(a1.nome, "Carlos");
a1.nota = 8.5;

JS (Objeto)

// Cria e preenche na hora
const aluno = {
    nome: "Carlos",
    nota: 8.5
};

// Ou depois:
aluno.turma = "B"; // Dinâmico!

26

Map e Filter

Formas poderosas de manipular listas sem usar for ou foreach.

const numeros = [2, 5, 10];

// Map: Cria nova lista modificada
const dobro = numeros.map(n => n * 2); 
// [4, 10, 20]

// Filter: Filtra a lista
const grandes = numeros.filter(n => n > 4); 
// [5, 10]
27

Assincronismo (Conceito)

No WinForms, se você roda um loop pesado, a janela trava ("Não respondendo").

O JS é assíncrono: ele manda fazer e continua a vida, evitando travar a tela.

28

Async / Await

Para lidar com tarefas demoradas (Banco de dados, Arquivos).

async function lerDados() {
    console.log("Iniciando...");
    
    // O código PAUSA aqui até o banco responder
    const dados = await bancoDeDados.buscar();
    
    console.log(dados);
}
