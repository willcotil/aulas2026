O Fluxo Sequencial: Um Caminho Só

Até agora, nossos programas eram como um trem em um trilho reto. O computador executava todas as linhas, de cima para baixo, sem pensar duas vezes.

1. Ler a Senha digitada
↓
2. Calcular Juros da Conta
↓
3. Exibir: Saldo é R$ 1.000.000

Qual é o problema aqui?

Se o usuário digitar a senha errada, o programa vai continuar andando em frente e exibirá o saldo da mesma forma! O programa é "cego", não avalia a situação.

Nós já somos mestres nisso!

O mundo real exige decisões a todo momento. Fazemos isso de forma tão natural que nem percebemos. Veja se reconhece a estrutura:

1. Se chover, fico em casa, senão, vou sair.
2. Se tiver dinheiro, compro um carro, senão, ando de Uber.
3. Se souber a questão, respondo, senão, deixo em branco.
4. Se o sinal estiver verde, atravesso, senão, espero.
5. Se estiver com fome, como uma maçã, senão, bebo água.
6. Se a nota for maior que 6, estou aprovado, senão, reprovado.
7. Se a bateria acabar, o celular desliga, senão, continua ligado.
8. Se hoje for sábado, durmo até tarde, senão, acordo às 6h.
9. Se eu for maior de 18 anos, posso tirar carteira, senão, ando de ônibus.
10. Se a senha estiver correta, acesso o sistema, senão, bloqueio!

A Condicional Simples: if (SE)

A palavra em inglês para "SE" é if. Ela cria um desvio no seu código.

O Esqueleto (Sintaxe Básica):

if ( condicao ) {
    // comandos a executar se for verdade
}

Como ler isso em português:

"SE a condição for verdadeira, ENTÃO execute os comandos que estão dentro das chaves."

O if na Prática

Vamos aplicar o esqueleto ao nosso problema do caixa eletrônico.

int senha;
printf("Digite sua senha numerica: ");
scanf("%d", &senha);

// Avaliando a condição
if (senha == 1234) {
    printf("Acesso liberado!\n");
    printf("Seu saldo e: R$ 1.000.000\n");
}

printf("Obrigado por usar nosso banco!\n");

Se o usuário digitar a senha errada, o computador pula o bloco protegido pelas chaves e não mostra o saldo, indo direto para a despedida.

A Pergunta Mágica

Dentro dos parênteses do if, nós colocamos uma comparação. O computador avalia se aquilo é verdade ou mentira usando os Operadores Relacionais.

if ( nota >= 6.0 )

  • 👉 >= significa Maior ou Igual a.
  • 👉 > significa Maior que.
  • 👉 < significa Menor que.
  • 👉 == significa Igual a (Cuidado: usamos DOIS sinais de igual para perguntar!).
  • 👉 != significa Diferente de.

O Plano B: else (SENÃO)

Muitas vezes, queremos que o programa faça algo se a condição for verdadeira, e outra coisa diferente se for falsa. Para isso existe o else.

O Esqueleto (Sintaxe Básica):

if ( condicao ) {
    // plano A (se for verdade)
} else {
    // plano B (se for mentira)
}

Como ler isso em português:

"SE a condição for verdadeira, ENTÃO faça o Plano A. SENÃO, faça o Plano B."

O else na Prática

Veja como o código decide o destino da pessoa baseado na idade.

int idade = 16;

if (idade >= 18) {
    printf("Voce e maior de idade.\n");
    printf("Pode tirar a carteira de motorista!\n");
} else {
    printf("Ainda nao tem idade suficiente.\n");
    printf("Continue andando de onibus.\n");
}

Regra Absoluta: O computador NUNCA executa os dois caminhos. É sempre um ou o outro!

O Perigo Silencioso das Chaves { }

As chaves dizem ao computador onde começa e onde termina o "reino" do if. Em C, se você não usar chaves, o if só protege a PRIMEIRA linha logo abaixo dele.

❌ Sem Chaves (Perigoso)

if (idade >= 18)
  printf("Pode entrar na festa.\n");
  printf("Pegue uma bebida.\n");

A segunda mensagem será impressa SEMPRE, até para uma criança de 10 anos, porque ela está fora do IF invisível! Isso causa bugs terríveis.

✅ Com Chaves (Seguro)

if (idade >= 18) {
  printf("Pode entrar na festa.\n");
  printf("Pegue uma bebida.\n");
}

As duas mensagens estão agrupadas no mesmo bloco. Ou o computador faz as duas, ou não faz nenhuma.

Recomendação de Ouro: Acostume-se a SEMPRE colocar as chaves { } logo que abrir o if, mesmo que vá escrever apenas uma linha!

O Código Completo: O Radar de Trânsito

Juntando tudo num programa real. Um radar lê a velocidade e decide se multa ou não o condutor.

#include <stdio.h>

int main() {
    int velocidade;
    int limite = 80;

    printf("Digite a velocidade do carro: ");
    scanf("%d", &velocidade);

    // Estrutura de Decisão
    if (velocidade > limite) {
        printf("\nMULTADO! Voce ultrapassou %d km/h.\n", limite);
        printf("Valor da multa: R$ 150.00\n");
    } else {
        printf("\nBoa viagem! Continue dirigindo com seguranca.\n");
    }

    return 0;
}

Exercícios (Mão na Massa - Prática)

4. Maioridade Penal (ex4.c)

Crie um programa que peça a idade do usuário (usando scanf). Se a idade for maior ou igual a 18, exiba "Você já é maior de idade. Pode ser preso!". Senão, exiba "Você ainda é menor de idade.".

5. Aprovado ou Reprovado (ex5.c)

Peça ao usuário para digitar a sua nota final em Programação (um float). Se a nota for maior ou igual a 6.0, mostre uma mensagem de aprovação festiva. Senão, mostre uma mensagem de incentivo para estudar mais para a recuperação.

6. Verificador de Senha (ex6.c)

Defina uma senha secreta no seu código (ex: int senha_secreta = 1234;). O programa deve pedir para o usuário digitar uma senha. Se ele acertar (use o operador == para comparar), exiba "Acesso Concedido". Se errar, "Acesso Negado".

7. Qual é o Maior? (ex7.c)

Peça ao usuário para digitar dois números inteiros diferentes (ex: num1 e num2). Crie uma estrutura condicional que compare os dois usando if/else e imprima apenas o número que for o maior de todos.

8. Desafio Final: A Loja com Desconto (ex8.c)

Uma loja dá um desconto especial de 10% apenas nas compras cujo valor total seja acima de R$ 100,00.
Crie um programa que leia o valor total da compra. Se o valor for maior que 100, calcule o desconto (multiplicando o valor por 0.10), subtraia do total e exiba o novo valor final a pagar. Se não for maior que 100, exiba o valor original sem desconto.
