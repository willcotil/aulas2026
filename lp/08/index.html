A Matemática da Comparação

O que o if realmente avalia?

Na aula sobre Estruturas de Decisão, aprendemos que o if precisa de uma condição para decidir se entra ou não num bloco de código.

if (idade >= 18) {
    printf("Pode entrar!");
}

Mas como é que o computador sabe se "idade >= 18" é verdadeiro ou falso?

Verdadeiro e Falso em C

Nós usamos as palavras "Verdadeiro" e "Falso" ou "Sim" e "Não". Mas para o computador (especialmente na linguagem C clássica), tudo se resume a números.

0

Falso (False)

Para a linguagem C, o número zero significa Falso. Se uma condição der 0, o if não é executado.

1

Verdadeiro (True)

O C devolve o número um para dizer que uma afirmação é Verdadeira. O if executará o bloco.

Curiosidade: Na verdade, o C considera qualquer número diferente de zero (como 5, -10, 42) como Verdadeiro. Apenas o 0 é estritamente Falso.

Testando o if Cego

Se o if apenas procura por 0 ou 1, podemos colocar os números diretamente dentro dos parênteses!

// Isto vai imprimir SEMPRE!
if (1) {
    printf("Entrou aqui!\n");
}
// Isto NUNCA vai imprimir!
if (0) {
    printf("Nunca serei lido!\n");
}

Os Juízes do Código

Os Operadores Relacionais comparam dois valores. Se a afirmação matemática for correta, o operador devolve 1. Se for incorreta, devolve 0.

Símbolo Significado Exemplo Mental Resultado (0 ou 1)
> Maior que 8 > 5 1
< Menor que 10 < 3 0
>= Maior ou igual a 5 >= 5 1
<= Menor ou igual a 4 <= 9 1
== IGUAL A 7 == 7 1
!= DIFERENTE DE 5 != 5 0

O Maior Erro dos Iniciantes

É incrivelmente comum confundir a Atribuição com a Comparação em C.

Um sinal (=)

x = 5

É o Comando de Atribuição. Significa "RECEBE". Você está forçando a caixa X a guardar o valor 5.

Dois sinais (==)

x == 5

É uma PERGUNTA. Significa "É IGUAL A?". O computador olha para a caixa X, compara com 5, e responde 1 (Sim) ou 0 (Não).

Guardando a Resposta de uma Comparação

Se as comparações resultam em números (0 ou 1), isso significa que podemos guardar o resultado de uma comparação dentro de uma variável inteira normal!

int nota = 8;

// A expressão (nota >= 6) é avaliada primeiro.
// 8 é maior ou igual a 6? Sim (1). 
// O valor 1 é guardado na variável 'passou'.
int passou = (nota >= 6);

printf("O aluno passou? (1=Sim, 0=Nao): %d\n", passou);
// Saída: O aluno passou? (1=Sim, 0=Nao): 1

int a = 10;

Variável Valor Atual

E se eu quiser comparar Letras (char)?

O computador não entende letras, ele entende números. Cada letra tem um código numérico na tabela ASCII (ex: 'A' = 65, 'B' = 66).

Portanto, você pode usar os operadores relacionais com letras normalmente!

Igualdade

('S' == 'S')

Verdadeiro (1). Os códigos numéricos são iguais.

Ordem Alfabética

('Z' > 'A')

Verdadeiro (1). O código numérico de 'Z' é maior que o de 'A', indicando que vem depois no alfabeto.

Nota: O C diferencia maiúsculas de minúsculas. 'a' == 'A' é Falso (0), pois têm códigos numéricos diferentes na tabela ASCII (97 vs 65).

O Problema da Comparação Dupla

Na matemática normal, escrevemos 10 < x < 20 para dizer que x está entre 10 e 20. Na linguagem C, isso NÃO funciona do jeito que você espera!

Entenda como o C lê:

10 < x < 20

Se x = 5, o computador resolve da esquerda para a direita:

  1. Ele resolve primeiro 10 < 5. Isso é Falso, logo dá o número 0.
  2. Aí ele pega esse resultado (0) e resolve a próxima parte: 0 < 20.
  3. 0 é menor que 20? Sim! Então é Verdadeiro (dá 1).

Resultado: O C achou que 5 está entre 10 e 20!
(Para resolver isso corretamente, usaremos Operadores Lógicos AND/OR futuramente).
