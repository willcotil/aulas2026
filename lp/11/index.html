Slide 03

O Limite do IF / ELSE Clássico

Na última aula, vimos que o if / else cria uma bifurcação simples. É como uma estrada que se divide em exatamente duas: ou você vai para a esquerda, ou vai para a direita.

Apenas 2 Situações

  • Aprovado OU Reprovado.
  • Maior de idade OU Menor de idade.
  • Senha Correta OU Senha Incorreta.

Mas e se houver mais?

  • Aprovado, Recuperação, ou Reprovado? (3 caminhos)
  • Bebê, Criança, Adolescente, Adulto ou Idoso? (5 caminhos)
  • Menu de Opções com 10 escolhas?

O Desafio

O mundo real raramente é apenas "preto ou branco". Como escrevemos um código que lida com zonas cinzentas e múltiplas categorias sem se tornar uma bagunça?

Slide 04

A Solução Ruim: Vários IFs Separados

A primeira ideia de muitos alunos é colocar vários ifs um debaixo do outro. Funciona? Às vezes sim. É correto? Não.

Péssimo Desempenho
int nota = 9;

if (nota >= 7) {
    printf("Aprovado"); // Entra aqui!
}

// O computador CONTINUA testando desnecessariamente!
if (nota >= 5 && nota < 7) {
    printf("Recuperacao");
}

if (nota < 5) {
    printf("Reprovado");
}

Por que é mau? O computador faz todas as perguntas, mesmo que já tenha encontrado a resposta correta na primeira linha. Num sistema grande, isso deixa tudo muito lento.

Slide 05

A Solução Elegante: else if (Senão Se)

Combine o else com um novo if na mesma linha. Isso cria uma "escada" de condições organizadas.

(Ou use a seta )
A Escada Perfeita
if (nota >= 7.0) {
printf("Aprovado!\\n");
} else if (nota >= 5.0) { // Lê-se: "Senão, SE..."
printf("Recuperacao!\\n");
} else { // A rede de segurança
printf("Reprovado!\\n");
}

Memória

nota=?

Avaliador

Aguardando...

Terminal

> _

A Regra de Ouro: O computador testa de cima para baixo. APENAS O PRIMEIRO bloco verdadeiro é executado!

Slide 06

O else solitário: A Rede de Segurança

Você pode empilhar dezenas de else if na sua escada. Mas, no final, é altamente recomendado colocar um else sozinho.

Capturando Exceções

O que sobrou!

O else final diz: "Se o programa chegou até aqui e falhou em TODAS as perguntas lá de cima, caia aqui obrigatoriamente."

É a forma ideal de tratar erros, como quando um utilizador digita uma opção que não existe num menu.

Slide 07

Exemplo Prático: Menu de Jogo

Veja como aplicamos a escada para ler a escolha de um menu interativo.

jogo.c
#include <stdio.h>

int main() {
    int opcao;
    printf("--- MENU PRINCIPAL ---\\n");
    printf("1 - Novo Jogo\\n2 - Carregar Jogo\\n3 - Opcoes\\n0 - Sair\\n");
    printf("Escolha: ");
    scanf("%d", &opcao);

    if (opcao == 1) {
        printf("Iniciando nova aventura...\\n");
    } else if (opcao == 2) {
        printf("Carregando save game...\\n");
    } else if (opcao == 3) {
        printf("Abrindo definicoes...\\n");
    } else if (opcao == 0) {
        printf("A sair do jogo. Adeus!\\n");
    } else {
        // A nossa rede de segurança!
        printf("Opcao Invalida! Tente novamente.\\n");
    }
    return 0;
}

Desafios (Teoria e Prática Inicial)

Crie um arquivo .c para cada desafio, compile e teste no terminal.

  1. O Teste do Menu (Mental):
    Baseado no código do Menu do Jogo no slide anterior. O que acontece exatamente (na memória e na tela) se o usuário digitar a opção 9? Descreva por quais ifs o computador passa.
  2. A Ordem Importa (Mental):
    Um programador fez este código: if (peso >= 60) printf("Leve"); else if (peso >= 90) printf("Pesado");. O lutador tem 100kg. O que vai imprimir? Por que a ordem dos ifs está errada?
  3. Categorias de Idade (Prática):
    Peça a idade de uma pessoa e classifique-a:
    - Menor que 13: Criança
    - Entre 13 e 17 (inclusive): Adolescente
    - Entre 18 e 59 (inclusive): Adulto
    - Maior ou igual a 60: Idoso
Slide 10

Desafios (Avançados)

  1. Calculadora Segura (ex4.c):
    Peça dois números decimais (float). Depois exiba o menu: "1. Somar | 2. Subtrair | 3. Multiplicar | 4. Dividir". Leia a opção escolhida. Use a escada de else if para fazer a conta certa e imprimir o resultado.
    Atenção: Na operação de divisão, adicione um if/else interno para verificar se o segundo número é zero. Se for, exiba um erro, caso contrário, faça a divisão. Adicione um else no final para opções inválidas do menu.
  2. O Radar de Trânsito:
    Leia a velocidade de um carro.
    - Até 80 km/h: "Velocidade Permitida".
    - De 81 a 100 km/h: "Multa Leve".
    - Acima de 100 km/h: "Multa Gravíssima!".
  3. Desafio Supremo: Imposto de Renda:
    Leia um salário (float). Calcule e exiba o desconto de imposto baseado na tabela:
    - Até R$ 2.000: Isento (0%).
    - De R$ 2.000,01 a R$ 5.000: Desconto de 10%.
    - Acima de R$ 5.000: Desconto de 20%.
    Mostre na tela: O Salário Bruto, O Valor Descontado e O Salário Líquido.
