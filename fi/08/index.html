01

Monitoramento Gráfico 🖱️

O Painel de Controle do Linux

Assim como o "Ctrl+Alt+Del" do Windows, a maioria das distribuições Linux de mesa vem com um aplicativo chamado Monitor do Sistema.

Ele é lindo, intuitivo e nos mostra exatamente o que está acontecendo por baixo do capô.

Monitor do Sistema: Processos

A primeira aba nos mostra a lista de todos os Processos (aplicativos e rotinas do sistema) abertos.

  • ✅ Podemos ver o nome do programa.
  • ✅ Quanto de CPU (%) ele está sugando.
  • ✅ Quanto de Memória RAM ele está ocupando.
  • Botão Direito: Podemos clicar em "Finalizar" se ele travar!
[Imagem: Captura de tela do System Monitor do Ubuntu na aba "Processos"]

Monitor do Sistema: Recursos

[Imagem: Captura de tela do System Monitor mostrando os gráficos coloridos de CPU, RAM e Rede]

A aba de Recursos exibe gráficos coloridos em tempo real da saúde da máquina.

  • Gráfico 1: Histórico de uso de cada núcleo da CPU.
  • Gráfico 2: Consumo de Memória RAM e Swap.
  • Gráfico 3: Tráfego da Placa de Rede (Download e Upload).

Gestão de Espaço: Analisador de Discos

"Meu PC está cheio, o que está ocupando tanto espaço?"

Aplicativos como o Analisador de Uso de Disco (Baobab) varrem o HD inteiro e criam um gráfico circular mostrando quais pastas estão mais "gordas". É perfeito para fazer faxina.

02

O Lado Terminal (CLI) 💻

Por que complicar se existe o Gráfico?

Se você for administrar um Servidor de verdade (Google, Netflix, Bancos), o computador fica trancado numa sala gelada. Não tem monitor ligado nele, não tem mouse e não tem interface gráfica!

Você acessa a máquina de longe usando apenas o Terminal. Vamos aprender os comandos de raio-X.

Comando: top

O clássico absoluto

O `top` mostra uma lista em tempo real dos processos (programas) que estão rodando e sugando recursos.

top - 14:20:00 up 2:10, 1 user, load average: 0.15, 0.05, 0.01
Tasks: 120 total, 1 running, 119 sleeping
%Cpu(s): 2.5 us, 0.5 sy, 0.0 ni, 97.0 id
MiB Mem : 8000.0 total, 3000.0 free, 2500.0 used

 PID USER PR NI VIRT RES SHR S %CPU %MEM TIME+ COMMAND  1234 root 20 0 1500000 200000 50000 S 5.5 2.5 0:15.00 mysql
 5678 aluno 20 0 2000000 500000 60000 R 2.0 6.2 1:20.00 chrome
    1 root 20 0 100000 10000 5000 S 0.0 0.1 0:05.00 init

Entendendo o top

PID (Process ID)

É o RG do programa! O sistema não conhece nomes, ele conhece esse número único gerado quando o app abriu.

USER

Quem abriu este programa? Foi você, ou foi o Root (sistema)?

%CPU / %MEM

A porcentagem de processador e memória RAM que este aplicativo está consumindo neste instante.

Para SAIR dessa tela, pressione q (quit).

Comando: htop

O top com grife e cores (muito mais fácil de ler)

Faz a mesma coisa que o top, mas desenha gráficos de barra coloridos direto no terminal usando símbolos de texto!

Além disso, ele permite que você navegue pela lista de processos usando as setinhas do teclado para cima e para baixo.

1 [|||||---------------------- 15.2%]
2 [|||||||||------------------ 32.1%]
Mem[||||||||||||||||--------- 2.5G/8.0G]
Swp[-------------------------- 0K/2.0G]

PID USER PRI NI VIRT RES SHR S CPU% MEM% TIME+ Command  5678 aluno 20 0 2000M 500M 60000 R 2.0 6.2 1:20.00 chrome

Comando: free

Como está a minha RAM?

Mostra uma foto instantânea do total de Memória RAM livre e usada no sistema.

DICA DE OURO: Sempre use a flag -h (Human Readable). O Linux adora mostrar números em bytes. O -h converte tudo para Mega (M) e Giga (G)!

aluno@meu-pc:~$ free -h
              total       used       free     shared
Mem:           7.8G       2.4G       3.1G      120M
Swap:          2.0G         0B       2.0G
aluno@meu-pc:~$ _

Comandos: df e du

Monitorando o HD / SSD pelo texto

df -h

(Disk Free) Mostra o espaço livre e ocupado de todas as **partições do HD inteiro**.

$ df -h
Sist. Arq.  Tam. Usado Disp.
/dev/sda1   250G  150G  88G

du -sh

(Disk Usage) Você coloca o caminho de uma pasta na frente, e ele calcula o peso **daquela pasta específica**.

$ du -sh /home/aluno/Musicas
15G   /home/aluno/Musicas

Raio-X Rápido 🎓

Você digitou free -h e viu que a RAM "usada" está quase batendo no limite "total", e a máquina começou a engasgar.

Qual comando você usaria logo em seguida para descobrir quem está comendo toda a sua RAM?

03

Matando Processos 💀

Travou tudo! E agora?

Você abriu o htop, encontrou aquele aplicativo problemático engolindo 100% da CPU e o mouse nem mexe direito.

É hora de pedir para o Kernel "assassinar" esse processo teimoso.

Comando: kill

Usando o RG do aplicativo

Lembra do PID que vimos lá no comando `top`? É o número de identificação único daquele programa.

Basta digitar `kill` seguido do PID.

Se ele recusar fechar amigavelmente, adicionamos a flag brutal -9 (Forçar Encaminhamento Imediato).

aluno@meu-pc:~$ # Suponha que o Chrome travado tem o PID 5678
aluno@meu-pc:~$ kill 5678
# Se o Chrome ignorar o aviso...
aluno@meu-pc:~$ kill -9 5678
[1]+ Morto chrome
aluno@meu-pc:~$ _

Comando: killall

Quando você não sabe o PID...

O `killall` permite que você encerre o programa usando o Nome dele, e ele matará todas as instâncias e janelas daquele aplicativo de uma vez.

aluno@meu-pc:~$ killall firefox
# Todas as 20 abas do Firefox travadas se fecham instantaneamente.
aluno@meu-pc:~$ _
04

Redes Básicas 🌐

Estamos Vivos na Internet?

Computadores sem rede são praticamente calculadoras glorificadas hoje em dia. Vamos aprender dois comandos rápidos para diagnosticar se o cabo de internet da nossa máquina está funcionando.

O que é um Endereço IP?

🏠 192.168.0.10

É como o CEP e Número da sua casa, só que no mundo digital.

Sem um número IP, as informações da internet (como as mensagens do WhatsApp) não sabem para onde ser entregues pelo roteador.

Comando: ip a

Descobrindo o seu endereço

Lista todas as placas de rede do computador (Cabo, Wi-Fi) e mostra os endereços IP que elas pegaram do roteador.

aluno@meu-pc:~$ ip a
1: lo: <LOOPBACK,UP,LOWER_UP> ...
2: eth0: <BROADCAST,UP> ...
    link/ether a1:b2:c3:d4:e5:f6
    inet 192.168.1.50/24 brd ...
aluno@meu-pc:~$ _

Comando: ping

Tem alguém aí?

Envia pequenos "pacotes" de dados para um destino na internet e vê se eles respondem de volta (eco). Se responderem, você tem internet!

aluno@meu-pc:~$ ping google.com
PING google.com (142.250.218.238) 56(84) bytes of data.
64 bytes from gru11s15-in-f14.net: icmp_seq=1 time=12.4 ms
64 bytes from gru11s15-in-f14.net: icmp_seq=2 time=12.1 ms
^C (Ctrl+C pressionado para parar)
--- google.com ping statistics ---
2 packets transmitted, 2 received, 0% packet loss

⚠️ No Linux o ping envia testes para sempre! Pressione Ctrl + C no teclado para forçar a parada.

Utilitários Extras 🧰

Comandos de "qualidade de vida" que facilitam muito a nossa rotina na tela preta.

clear e history

clear

Limpa toda a bagunça visual da tela do terminal, deixando o prompt lá no topo limpinho de novo. (Atalho: Ctrl + L)

history

Mostra uma lista numerada de todos os comandos que você já digitou no passado. Ótimo pra lembrar aquele comando longo!

Informações do Sistema: uname -a

Mostra rapidamente detalhes básicos do sistema, a versão do Kernel Linux, a arquitetura da máquina e a data de compilação.

aluno@meu-pc:~$ uname -a
Linux meu-pc 5.15.0-76-generic #83-Ubuntu SMP Wed Jun 21 20:23:31 UTC 2023 x86_64 x86_64 x86_64 GNU/Linux
aluno@meu-pc:~$ _

Dica Hacker: O comando neofetch é mais bonito e desenha logos na tela, mas ele requer a instalação de pacotes de fora, o uname já vem instalado em tudo!

Cheat Sheet de Hoje (Sua colinha)

Comando Para que serve?
top / htop Gerenciador de tarefas (Processos vivos e CPU/RAM).
free -h Mostra a quantidade de Memória RAM livre/usada.
df -h Mostra o espaço livre no HD inteiro (partições).
du -sh [pasta] Calcula o "peso" (tamanho) de uma pasta específica.
kill / killall Encerra um programa travado à força (-9).
ip a / ping Descobre seu IP e testa conexão com a internet.
