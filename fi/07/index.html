A Grande Biblioteca 📚

O HD/SSD é apenas um pedaço de silício ou metal cheio de 0s e 1s espalhados.

Sistema de Arquivos (File System)

É o "índice" do Sistema Operacional. Ele organiza essa bagunça em Pastas (Diretórios) e Arquivos para que a gente consiga achar as coisas.

No Windows usamos NTFS. No Linux, usamos geralmente o EXT4.

Regra de Ouro do Linux

"Tudo é um arquivo!"

No Linux, um texto é um arquivo, uma foto é um arquivo, e ATÉ O SEU MOUSE E TECLADO são representados como arquivos de texto dentro do sistema.

A Árvore de Diretórios 🌳

No Windows, as coisas começam no Disco C:\. No Linux, começam na Raiz (/).

Onde ficam seus arquivos

Caminhos: Como chegar no seu quarto?

Caminho Absoluto

É como dar o seu endereço completo: País, Estado, Cidade, Rua, Número.

/home/aluno/Documentos/texto.txt

Começa sempre da raiz ( / ) e não tem erro, funciona de qualquer lugar que você esteja.

Caminho Relativo

É como dizer: "Vá para o quarto ao lado". Depende de onde você está agora.

Documentos/texto.txt

Só funciona se você já estiver dentro da pasta "/home/aluno". Não tem a barra inicial.

O Temido Terminal

Esqueça o mouse. A partir de agora, vamos falar com o computador escrevendo. É mais rápido, mais poderoso e faz você parecer um hacker nos filmes. 🕶️💻

O Prompt (A linha de espera)

Sempre que você abre o terminal, ele mostra algo parecido com isso:

aluno@meu-pc:~$ _
aluno@meu-pc Quem sou eu? (Usuário) e em qual máquina estou logado.
~ (Til) Onde eu estou? O Til é um atalho que significa "A minha pasta pessoal" (/home/aluno).
$ (Cifrão) Sou um usuário comum. (Se fosse um #, eu seria o Administrador).

A Anatomia de um Comando 🦴

Antes de sairmos digitando, precisamos entender como o Linux lê o que escrevemos. Quase tudo segue uma "receita" básica de 3 partes:

ls
-l -a
/home/aluno

1. O Comando

"O Verbo / A Ação"

É o nome do programa. É a ação principal que você quer fazer.

Ex: listar, copiar, apagar.

2. As Flags (Opções)

"O Advérbio / O Modo"

Começam com um traço (-). Elas modificam como o comando age.

Ex: listar de forma detalhada (-l).

3. O Argumento

"O Alvo / O Objeto"

Informa ao comando onde ou em quê ele deve agir.

Ex: listar os arquivos "desta pasta".

💡 Nota Ninja: O Comando é obrigatório, mas as Flags e os Argumentos costumam ser opcionais!

Comando: pwd

Print Working Directory (Onde diabos eu estou?)

Às vezes a gente se perde andando pelas pastas. O `pwd` imprime na tela o Caminho Absoluto de onde você está neste exato momento.

Pense nele como o GPS do terminal que te mostra o seu pino no mapa da árvore de diretórios.

aluno@meu-pc:~$ pwd
/home/aluno/Imagens/Ferias
aluno@meu-pc:~/Imagens/Ferias$ _

Comando: cd

Change Directory (Andando pela casa)

Você digita `cd` seguido do nome da pasta (Argumento) para entrar nela.

Atenção: Se o nome da pasta tiver espaços, você precisa usar aspas (Ex: cd "Minhas Musicas").

aluno@meu-pc:~$ cd Documentos
aluno@meu-pc:~/Documentos$ pwd
/home/aluno/Documentos
aluno@meu-pc:~/Documentos$ _

Atalhos Ninjas do cd 🥷

Porque digitar o caminho todo cansa...

cd ..

Sobe um nível. Sai do quarto e vai pro corredor.

cd ~

Leva você direto para o seu quarto (Sua pasta /home/usuario).

cd /

Leva você como um foguete para a Raiz de todo o sistema.

cd -

É o botão "Voltar". Joga na última pasta em que você estava.

Minijogo: Navegando no Escuro 🚶‍♂️

Você está em: /home/aluno/Musicas

Onde você vai parar se digitar o comando abaixo?

cd ..

Comando: ls

List (O que tem aqui dentro?)

É o equivalente a dar dois cliques numa pasta no Windows para ver o que tem dentro. Ele lista arquivos e diretórios.

Dica: No Linux, pastas geralmente aparecem coloridas em azul, e arquivos em branco ou verde (se forem executáveis).

aluno@meu-pc:~$ ls
Documentos Downloads tarefa.txt
aluno@meu-pc:~$ _

Turbinando o ls (As Flags)

Usando opções para extrair mais segredos do Kernel.

O ls puro só mostra os nomes básicos. Mas veja o que acontece se adicionarmos flags:

ls -a (All): Mostra até os "arquivos fantasmas" do Linux (arquivos ocultos que começam com ponto, ex: .config).

ls -l (Long): Cria uma lista detalhada. Mostra tamanho, data e uma sopa de letrinhas estranhas (rwxr-x---) no começo da linha.

O que diabos é rwxr-x---? Vamos descobrir agora!

aluno@meu-pc:~$ ls -l
-rw-r--r-- 1 aluno familia 4096 Out 10 14:00 senha.txt
aluno@meu-pc:~$ _

Permissões no Linux 🛡️

O Problema do Irmão Mais Novo

Você divide o computador com seu irmãozinho. Ele loga na conta dele, vasculha suas pastas e acidentalmente apaga o save do seu jogo de 100 horas.

No Linux, isso é impossível se as Permissões estiverem corretas!

Quem é quem? (Usuário, Grupo, Outros)

Para cada arquivo, o Linux divide o mundo em três categorias de pessoas:

👤

Dono (User)

A pessoa que criou o arquivo. É a "dona do diário".

👥

Grupo (Group)

A "família" ou "equipe". Pessoas que têm acesso VIP.

👽

Outros (Others)

Os penetras. O resto do mundo que acessa o sistema.

Os 3 Poderes Mágicos (r w x)

r

Read (Ler)

Pode abrir o arquivo, ler o texto, ou ver o conteúdo de uma pasta.

w

Write (Escrever)

Pode modificar o arquivo, salvar por cima ou apagar.

x

eXecute (Executar)

Se for um programa/jogo, pode rodar. Se for uma pasta, pode entrar (`cd`) nela.

E esse tracinho ( - ) ?

- - -

O tracinho significa PERMISSÃO NEGADA.

O Linux sempre espera a ordem de 3 letras pra cada pessoa (r w x). Se alguém não tem aquele poder, ele preenche com um traço pra não perder a posição correta na leitura.

A Sopa de Letrinhas Resolvida

O Linux junta os 3 Poderes (rwx) para os 3 Grupos (User, Group, Others), criando um código de 9 caracteres de segurança.

rwx r-x r--

Dono

Lê, Grava, Executa

Grupo

Lê, Negado (-), Executa

Outros

Só Lê. (Penetras!)

De volta ao ls -l

Fim do mistério: Estamos olhando os crachás!

Agora que você conhece a anatomia da segurança, a lista detalhada do ls -l finalmente faz todo o sentido:

$ ls -l
-rw-r--r-- 1 aluno familia 4096 Out 10 14:00 senha.txt
drwxr-xr-x 2 aluno familia 4096 Out 10 14:05 Downloads
-rwxrwxrwx 1 aluno familia 8192 Out 10 14:10 jogo.exe

Nota: O primeiríssimo caractere (antes do rwx) indica se é Diretório (d) ou Arquivo (-).

Desvendando a linha (Anatomia)

-rw-r--r-- 1 aluno familia 4096 texto.txt
👤 Dono: "aluno"
Permissão: rw- (Lê e Grava)
👥 Grupo: "familia"
Permissão: r-- (Só lê)
👽 Outros: O mundo
Permissão: r-- (Só lê)

Decifrador de Permissões 🕵️‍♀️

Você vê o seguinte arquivo na pasta: -rwxr-x---

O que o grupo "Outros" (os penetras) pode fazer com esse arquivo?

Cheat Sheet (Sua colinha)

Comando Tradução Mental O que faz?
pwd Onde estou? Mostra o caminho absoluto da pasta atual.
ls Lista Mostra os arquivos na pasta.
cd Mudar pasta Entra em um diretório. (cd .. para voltar).
cd ~ Ir pra Casa Volta instantaneamente para a sua pasta de usuário.
ls -l Lista Longa Mostra os arquivos com detalhes, incluindo as permissões!

O Cuidado de Ser Root (Deus)

O usuário 'root' (administrador) ignora todas as regras de permissão (o tracinho - não funciona com ele).

Ele pode ler, escrever e apagar qualquer coisa, inclusive o próprio Kernel do sistema. É por isso que você nunca usa o PC logado como Root. Você só usa o comando sudo quando precisa de poderes temporários.

E se eu tentar apagar arquivo dos outros?

Se você for um mero mortal (usuário comum) e não tiver o 'w' (Write) configurado no seu grupo, o terminal cuspirá na sua cara de forma muito direta e fria:

bash: arquivo.txt: Permission Denied

E não adianta chorar, o Kernel não tem sentimentos.
