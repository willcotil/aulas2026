A Máquina "Burra"

Lembre-se: O Processador só entende duas coisas: 0 e 1 (Tensão Baixa e Alta).

Se você comprar as melhores peças do mundo e montá-las, o que acontece ao ligar a tomada?

ABSOLUTAMENTE NADA.

O Hardware precisa de instruções precisas para realizar qualquer tarefa.

Fase 1: Linguagem de Máquina

Nos primórdios (ENIAC), os programadores precisavam escrever o código diretamente em binário ou conectando cabos físicos.

Imagine ter que escrever 100 páginas de zeros e uns só para fazer o computador somar 2 + 2. O índice de erro era gigantesco.

01010111 01100101 01101100 01100011 01101111 01101101 01100101 00100000 01110100 01101111 00100000 01001000 01100101 01101100 01101100

Fase 2: Assembly (A 1ª Abstração)

MOV AX, 5
ADD AX, 3
INT 80h

Criamos códigos mnemônicos (palavras curtas) para substituir os números binários. Um programa chamado Assembler traduzia essas palavras para 0 e 1.

Facilitou, mas ainda era terrível: cada tipo de processador tinha um Assembly diferente. O código feito para Intel não rodava em AMD.

Fase 3: Linguagens de Alto Nível

Criamos linguagens que parecem com o inglês (C, Java, Python). Elas são traduzidas para a máquina por Compiladores.

if (nota >= 6) {
  print("Aprovado");
}

O Grande Problema: Cada programa que você escrevia tinha que ter um código gigante para controlar a impressora, a tela, o teclado, a memória... Era reinventar a roda toda vez!

O Nascimento do S.O.

Por que não criar um único programa mestre que saiba conversar com todas as peças do hardware?

Assim, se eu criar um jogo, eu não preciso programar como o jogo acende o monitor. O jogo apenas "pede" para o programa mestre: "Ei, desenha esse boneco na tela pra mim?"

Este programa mestre é o Sistema Operacional (OS).

As Camadas de Abstração

Usuário (Você)
Aplicações (Chrome, Excel, Jogos)
SISTEMA OPERACIONAL
Firmware / Drivers
Hardware (CPU, RAM, Discos)

O S.O. fica no meio. Ele protege o hardware de usuários inexperientes e protege os programas das complexidades eletrônicas da máquina.

Entrando no Kernel

O Sistema Operacional é gigante (calculadora, interface gráfica, relógio). Mas existe um "núcleo" duro, que tem poder absoluto sobre a máquina.

KERNEL

O que é o Kernel?

[Imagem: Ilustração de uma Semente/Caroço (Kernel significa núcleo/semente em inglês)]

O Kernel é o componente central do S.O. Ele é o primeiro programa a ser carregado na RAM (após o boot da placa-mãe) e fica lá ativo até o PC desligar.

Poder Absoluto:

Ele é a única coisa no computador que tem permissão para acessar a CPU, a RAM e o Hardware diretamente e sem restrições.

Funções do Kernel: 1. Processos

O computador tem 1 processador, mas você escuta Spotify, usa o Chrome e baixa um arquivo ao mesmo tempo.

Quem faz a mágica? O Kernel! Ele fatia o tempo do processador em milissegundos e distribui entre os programas. Ele troca as tarefas tão rápido que parece simultâneo (Multitarefa Preemptiva).

Funções do Kernel: 2. Memória RAM

O Porteiro da RAM

O Chrome pede 2GB de RAM. O Kernel procura espaço vazio, reserva e entrega ao Chrome.

Isolamento: Ele garante que o Chrome não leia a área de memória onde está a sua senha do Banco digitada em outro app. Segurança em primeiro lugar.

Memória Virtual (Paging)

Acabou a RAM física? O Kernel pega um pedaço do HD/SSD e "finge" que é RAM para o PC não travar. O sistema fica mais lento, mas não crasha (Página de Swap).

Funções: Arquivos e Hardware

Sistemas de Arquivos

O HD é só um mar de magnetismo ou transistores. O Kernel organiza isso em Pastas, Arquivos, e gerencia as permissões de leitura/escrita (ex: NTFS no Windows, EXT4 no Linux).

Device Drivers (E/S)

O Kernel usa pequenos programas chamados Drivers para saber exatamente como conversar com a placa de vídeo, a impressora, a placa de rede e o mouse de marcas diferentes.

A Divisão de Poder (Anéis de Proteção)

Para o PC não explodir com qualquer aplicativo mal feito, o Processador possui níveis de privilégio de segurança (Rings).

Ring 3 (User Space)
Ring 1 & 2
(Drivers)
Ring 0
KERNEL

Seus jogos e apps rodam no Ring 3 (Modo Usuário). Eles não podem tocar na memória física. Se tentarem, o Kernel (Ring 0) intervém e "mata" o programa (Fecha sozinho).

System Calls (Syscalls)

Se o aplicativo (Ring 3) não pode acessar o disco, como você salva um arquivo no Word?

Ele faz um pedido oficial e educado chamado System Call.

O Word diz: "Sr. Kernel, por favor, salve estes dados no disco Rígido." O Kernel verifica se o usuário tem permissão, faz o trabalho de forma segura, e avisa o Word que terminou com sucesso.

Analogia: O Restaurante 🍽️

👤

Não pode entrar na cozinha.

📝

Leva a mensagem autorizada.

👨‍🍳

O Kernel (Ring 0).

O único que acessa a despensa e prepara o prato.

🥩

Despensa/Fogão

O Hardware (RAM/HD).

Os recursos físicos brutos.

Quando o Chef desmaia...

Se o Chrome trava (Modo Usuário), você aperta `Ctrl+Alt+Del` e o fecha. O Kernel continua vivo administrando o resto.

Mas e se o problema for no Kernel ou em um Driver essencial (Ring 0)?

O sistema inteiro para imediatamente para evitar que o Hardware seja danificado fisicamente ou que os dados do HD sejam corrompidos permanentemente.

03

Sistemas Operacionais Modernos

A Guerra das Filosofias: O Fechado, o Aberto e o Bonito.

Microsoft Windows

  • Modelo: Código Fechado (Proprietário). Licença paga.
  • Kernel: Windows NT (Híbrido).
  • Foco: Interface Gráfica (GUI) integrada profundamente. Foco extremo na facilidade de uso para o usuário final doméstico e retrocompatibilidade.
  • Domínio: Domina cerca de 70% do mercado de computadores Desktop. É a plataforma principal e absoluta para Gamers e softwares corporativos padronizados (Office, Adobe).

A Evolução do Windows

[Imagem: Windows 95 interface clássica com o botão Iniciar]

O primeiro Windows (1985) não era um Sistema Operacional completo, era apenas uma "casca gráfica" que rodava em cima do MS-DOS (sistema de linha de comando preta).

Apenas nos anos 90, e consolidadamente com o Windows XP, a Microsoft desenvolveu um Kernel moderno, robusto e com multitarefa real que não dependia mais do DOS como base.

GNU / Linux

  • Modelo: Código Aberto (Open Source / Licença GPL). Gratuito e feito por milhares de colaboradores globais.
  • Kernel: Linux (Monolítico, criado pelo estudante Linus Torvalds em 1991).
  • Foco: Estabilidade extrema, segurança, customização ilimitada e linha de comando (Terminal/CLI) poderosa.
  • Domínio: Domina a Internet! Roda em mais de 90% dos Servidores WEB do mundo, em 100% dos Supercomputadores e é o núcleo base do Android (Smartphones).

O que é uma Distro Linux?

"Linux" na verdade é só o nome do Kernel (o motor do carro). Para ter um sistema completo que você possa usar, você precisa adicionar o chassi, volante e bancos.

Apple macOS

Kernel: Darwin (Baseado em UNIX, o "avô" do Linux).

Modelo: Sistema Fechado, atrelado estritamente à venda de hardware exclusivo da Apple (MacBooks, iMacs).

Ele une a estabilidade feroz e os comandos de um terminal UNIX com a interface gráfica mais polida, intuitiva e controlada do mercado. Altamente adotado por designers, editores de vídeo e desenvolvedores de software.

A Interface de Linha de Comando (CLI)

No Windows doméstico, você faz quase tudo com o mouse (GUI - Interface Gráfica). No universo Linux profissional e de servidores, o poder verdadeiro reside no Terminal.

O Terminal (Shell) permite conversar diretamente com o sistema usando comandos de texto curtos, rápidos e precisos.

Por que aprender a usar o Terminal?

Ele é incomparavelmente mais rápido para tarefas repetitivas, permite a criação de Scripts (automação) e consome muito menos memória e processamento da máquina.
