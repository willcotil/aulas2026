Recapitulação: A Fórmula Mágica

1. Identificando a Rede e o Host

O IPv4 possui 32 bits. O prefixo (CIDR) indica quantos bits pertencem à REDE.

/26
26 bits para Rede → 6 bits para Host

2. Calculando a Capacidade

Com os bits de host encontrados (chamemos de n), usamos a fórmula para descobrir o total de endereços:

2n
  • Tamanho do Bloco: 2n
  • Hosts Úteis: 2n - 2 (Tiramos o IP da rede e o IP de broadcast).

Recapitulação: A Regra do Bloco

Como encontrar o Endereço de Rede a partir do tamanho do bloco?

O octeto "quebrado" sempre conta em múltiplos do tamanho do bloco.

Se o seu bloco tem tamanho 64, as redes vão pular de 64 em 64 naquele octeto:

.0 → .64 → .128 → .192
  • A Rede: É o múltiplo atual.
  • O Broadcast: É o número imediatamente anterior ao próximo múltiplo.
  • Os Hosts: Tudo que estiver no meio.

O Método em 4 Passos

Use este roteiro mental para resolver qualquer um dos exercícios a seguir:

1
Ache os bits de Host: Subtraia o prefixo de 32. Ex: 32 - /26 = 6.
2
Ache o Bloco (Mágica): Calcule 2 elevado aos bits de host (2n). Ex: 26 = 64.
3
Posicione o IP: Encontre em qual "fatia" do bloco o IP do problema se encaixa. Ex: O IP .70 está na fatia entre 64 e 128. Logo, a rede é 64.
4
Defina os limites: A Rede é o início do bloco. O Broadcast é o limite final (próximo bloco - 1). O útil é o meio.
Pegue papel e caneta. Os exercícios começam agora!

Exercício 1: Cálculo Básico

Enunciado: Um computador recebeu o endereço IP 192.168.1.10 /26.

Encontre os seguintes dados para esta máquina:

  • Endereço da Rede
  • Endereço de Broadcast
  • Faixa de IPs Úteis (Primeiro e Último válidos)

Solução

1
Bits de Host: 32 - 26 = 6
2
Tamanho do Bloco: 26 = 64
3
O octeto de host é "10". O valor 10 cai no primeiro bloco que vai de 0 até 63.
  • Rede: 192.168.1.0
  • Broadcast: 192.168.1.63
  • IPs Úteis: 192.168.1.1 até 192.168.1.62

Exercício 2: Onde estou?

Enunciado: Durante uma configuração de servidor, identificou o IP 10.0.5.150 /23.

Neste caso, a máscara "invadiu" o 3º octeto. Qual é o IP de Rede e o Broadcast deste servidor?

Solução

1
O prefixo /23 significa 23 bits de rede. A divisão cai no 3º octeto (que comporta os bits do 17 ao 24).
Bits de host totais: 32 - 23 = 9. Desses, 8 estão no 4º octeto e 1 está no 3º octeto.
2
No 3º octeto, sobrou 1 bit para host. Bloco: 21 = 2.
As redes pulam de 2 em 2 no terceiro octeto (0.0, 2.0, 4.0, 6.0...).
3
O IP é 10.0.5.150. O valor 5 cai no bloco que começa no 4 e vai até antes do 6.
  • Rede: 10.0.4.0
  • Broadcast: 10.0.5.255 (Último IP antes da rede 10.0.6.0)

Exercício 3: Planeamento de Rede

Enunciado: Uma empresa comprou a rede 192.168.10.0 /24. Eles precisam dividir essa rede para os departamentos. Cada departamento precisa de, no máximo, 30 computadores.

Qual é o prefixo CIDR (/?) ideal que o administrador deve escolher para não desperdiçar endereços?

Solução

1
Precisamos de acomodar até 30 IPs úteis. Usamos a fórmula: 2n - 2 ≥ 30.
2
Testando os valores de n (bits de host):
Se n=4 → 24 - 2 = 14 (Insuficiente)
Se n=5 → 25 - 2 = 30 (Exato!)
3
Se precisamos de 5 bits de host, quantos bits sobram para a rede?
32 - 5 = 27.

A resposta é a máscara /27. A rede seria dividida em blocos de 32.

Exercício 4: Identificação de Limites

Enunciado: Foi atribuído o IP 172.16.50.10 /20 a um router principal.

Atenção: Resolução para entrega no Classroom
1. Em que octeto atua esta máscara e qual é o tamanho do bloco?
2. Qual é o Endereço de Rede a que pertence este router?
3. Qual é o Endereço de Broadcast desta rede?

Dica: Use a fórmula 32 - 20 para achar os bits de host totais, divida entre o 3º e 4º octeto e calcule o tamanho do bloco no octeto alterado.

Exercício 5: Cálculo de Capacidade

Enunciado: Um colega configurou a máscara de um servidor Windows como 255.255.248.0.

Atenção: Resolução para entrega no Classroom
1. Qual é o valor equivalente em notação CIDR (Prefixo /x)?
2. Exatamente quantos Hosts Úteis podem existir numa sub-rede com esta configuração?

Dica: Converta o número 248 para binário para descobrir quantos bits a "1" existem no 3º octeto.

Exercício 6: Análise de Comunicação

Enunciado: O PC-A tem o IP 192.168.100.5 /27 e o PC-B tem o IP 192.168.100.35 /27. Eles estão conectados ao mesmo switch L2.

Atenção: Resolução para entrega no Classroom
Eles estão na mesma sub-rede ou em sub-redes diferentes? Se o PC-A enviar um "ping" diretamente para o PC-B sem a ajuda de um router (gateway), irá funcionar? Justifique com o tamanho do bloco.

Dica: Calcule primeiro o tamanho do bloco da máscara /27. Depois, veja em quais múltiplos (blocos) os valores 5 e 35 se encaixam.

Exercício 7: Engenharia de Redes

Enunciado: Tens disponível o enorme bloco de IPs 10.1.0.0 /16. O cliente pediu-te para criar um design onde suportes 500 sub-redes distintas, utilizando essa mesma faixa.

Atenção: Resolução para entrega no Classroom
Qual é a nova máscara (prefixo CIDR) que deverás aplicar para conseguir, pelo menos, 500 sub-redes? Quantos hosts vão sobrar por sub-rede no teu design?

Dica: Para calcular sub-redes, a fórmula é 2s ≥ X, onde 's' é o número de bits que vais "roubar" da porção de hosts para dar à porção de rede.

Exercício 8: O Último Host

Enunciado: Configuraste um router com o endereço IP 192.168.200.120 /29.

Atenção: Resolução para entrega no Classroom
1. Descobre a qual Rede pertence este IP.
2. Descobre qual é o último endereço IP configurável (útil) nesta exata sub-rede antes do endereço de broadcast.

Dica: A máscara /29 é muito restrita. Deixa quantos bits de host? Qual o tamanho do bloco? A que bloco pertence o valor 120?
"O cálculo de sub-redes não é adivinhação, é apenas matemática base 2. Dominar a Regra do Bloco transforma qualquer máscara num mapa óbvio da rede."
