O Problema: a Limitação do Switch

Nas aulas anteriores, vimos que o switch conecta computadores dentro de uma mesma rede local (LAN). Mas ele tem um limite claro:

🔌

O switch conhece...

Quais dispositivos estão conectados em suas portas, usando o endereço físico (MAC) de cada um.

🚫

O switch não sabe...

Como chegar a um dispositivo que está em outra rede — outra empresa, outro bairro, outro país.

🌐

A solução é...

O roteador — dispositivo que sabe como interligar redes diferentes e encaminhar dados entre elas.

Imagine que o switch é o sistema de ramais de um único escritório. O roteador é a central telefônica que conecta escritórios em cidades diferentes.

Switch e Roteador: A Diferença Fundamental

🔌 Switch

  • Opera na Camada 2 do Modelo OSI (Enlace)
  • Identifica dispositivos pelo endereço MAC (físico)
  • Conecta dispositivos dentro da mesma rede (LAN)
  • Aprende quais MACs estão em cada porta automaticamente
  • Não entende endereços IP — não sabe "onde fica" a internet
Analogia: interfone de um prédio — fala com quem está no mesmo prédio.

🌐 Roteador

  • Opera na Camada 3 do Modelo OSI (Rede)
  • Identifica redes pelo endereço IP (lógico)
  • Conecta redes diferentes entre si (LAN ↔ internet)
  • Mantém uma tabela de rotas com caminhos conhecidos
  • Decide o melhor caminho para cada pacote de dados
Analogia: agência dos Correios — sabe para qual cidade e estado mandar cada encomenda.

O Roteador: O Carteiro da Internet

"Um roteador é um dispositivo de rede que recebe pacotes de dados e decide qual o melhor caminho para encaminhá-los até o destino."

📦

1. Recebe

Um pacote chega com um IP de destino no cabeçalho

🗺️

2. Consulta

Busca o destino na sua tabela de rotas

🚀

3. Encaminha

Envia o pacote pela interface correta em direção ao destino

Dado curioso: um pacote do Brasil para o Japão pode atravessar 15 a 20 roteadores antes de chegar — em menos de 300 milissegundos.

Tipos de Roteadores

Nem todo roteador é igual — o porte e a capacidade variam muito dependendo do uso:

🏠

Doméstico

O roteador Wi-Fi da sua casa. Conecta sua rede local (computadores, celulares) à internet via ISP. Geralmente já tem switch e ponto de acesso Wi-Fi embutidos.

Ex.: TP-Link, Intelbras
🏢

Corporativo

Usado em empresas e universidades. Suporta muitos usuários simultâneos, VPNs, QoS (qualidade de serviço) e configurações avançadas de segurança.

Ex.: Cisco, Juniper, MikroTik
🌐

Núcleo (Core)

Roteadores da espinha dorsal da internet. Processam bilhões de pacotes por segundo e mantêm rotas para todas as redes do planeta.

Ex.: Cisco CRS, Juniper PTX

Roteamento: Encontrar o Melhor Caminho

Definição: Roteamento é o processo de determinar o caminho que um pacote de dados deve seguir desde a origem até o destino, passando por uma ou mais redes intermediárias.

🛰️ Analogia com GPS

  • Você digita o destino → GPS calcula a rota
  • O pacote carrega o IP de destino → o roteador calcula o próximo passo
  • O GPS recalcula em caso de congestionamento → protocolos dinâmicos fazem o mesmo
  • Você pode ter preferências (mais rápido, menos pedágio) → o roteador tem métricas

📡 Como o roteamento funciona?

  • O pacote viaja de roteador em roteador
  • Cada roteador toma sua própria decisão: "para onde passo esse pacote agora?"
  • Nenhum roteador precisa saber o caminho completo — só o próximo passo
  • É como pedir direção na rua: cada pessoa te indica o próximo trecho

Contar saltos é útil! O comando traceroute (ou tracert no Windows) mostra cada roteador no caminho e o tempo que o pacote levou em cada salto.
Mais saltos nem sempre = mais lento. A velocidade do link entre dois roteadores importa mais do que o número de saltos. Um link de fibra com 10 saltos pode ser mais rápido que um link de satélite com 3 saltos.

TTL: A Data de Validade do Pacote

Imagine um pacote que, por algum erro de configuração, fica circulando entre roteadores para sempre — sem nunca chegar ao destino. Para evitar isso, existe o TTL (Time to Live).

⏱️ Como o TTL funciona?

  • Quando o pacote é criado, recebe um TTL inicial (geralmente 64 ou 128)
  • A cada roteador pelo qual passa, o TTL é decrementado em 1
  • Se o TTL chegar a zero, o roteador descarta o pacote
  • O roteador envia uma mensagem de erro de volta à origem

🎯 Exemplo prático

  • Pacote sai com TTL = 64
  • Passa pelo Roteador 1 → TTL vira 63
  • Passa pelo Roteador 2 → TTL vira 62
  • ... e assim por diante
  • Após 64 saltos sem encontrar o destino → descartado
O traceroute usa o TTL de forma inteligente: envia pacotes com TTL=1, TTL=2, TTL=3... Cada vez que um roteador descarta o pacote, ele responde revelando sua identidade — assim o traceroute "mapeia" o caminho!

O que é uma Rota?

Uma rota é uma instrução que diz ao roteador: "para chegar à rede X, encaminhe o pacote pelo caminho Y."

🗺️ Componentes de uma Rota

  • Destino: qual rede ou endereço o roteador quer alcançar
  • Gateway (próximo salto): o endereço IP do roteador vizinho que sabe chegar lá
  • Interface: qual porta física do roteador deve ser usada para sair
  • Métrica: custo ou qualidade dessa rota (quanto menor, melhor)

📌 Analogia dos Correios

Uma agência dos Correios tem uma instrução interna:

  • Destino: encomendas para São Paulo
  • Próximo passo: entregar ao caminhão da Rodovia Anhanguera
  • Saída: doca número 3

Esse é exatamente o raciocínio de uma entrada na tabela de rotas.

A Tabela de Rotas: o Mapa do Roteador

O roteador consulta sua tabela de rotas para cada pacote que recebe. Veja um exemplo simplificado:

DestinoGateway (Próximo Salto)Interface de SaídaTipo
Rede local (192.168.1.x) — conexão direta — eth0 (LAN) Direta
Rede da filial (10.0.0.x) 192.168.1.1 eth0 Estática
Qualquer outro destino 192.168.1.254 eth1 (WAN) Padrão (Default)
Leitura da tabela: o roteador verifica o IP de destino do pacote e procura a linha mais adequada. Se nenhuma linha específica corresponder, usa a rota padrão (última linha) — que aponta para o provedor de internet.

O Gateway Padrão: a Porta de Saída

O gateway padrão (default gateway) é o roteador que um dispositivo usa quando quer se comunicar com qualquer rede fora da sua rede local.

🏠 No seu computador em casa

  • Seu PC tem um IP local (ex: 192.168.1.10)
  • O roteador Wi-Fi tem o IP (ex: 192.168.1.1)
  • Esse IP do roteador é configurado como o gateway padrão do seu PC
  • Quando você acessa qualquer site, seu PC manda o pacote para 192.168.1.1 primeiro

🤔 Como o PC decide?

Antes de qualquer coisa, o computador se pergunta:

  • "O destino está na minha rede local?"
  • Se SIM → entrega diretamente (via switch)
  • Se NÃO → manda para o gateway padrão
Sem um gateway padrão configurado, o computador só consegue se comunicar dentro da rede local.

Como o Roteador Escolhe o Caminho?

O roteador usa métricas para comparar rotas e decidir a melhor opção quando há mais de um caminho possível:

🔢 Número de Saltos (Hop Count)

Prefere o caminho com menos roteadores no meio. Simples, mas ignora velocidade dos links. Usado pelo protocolo RIP.

📡 Largura de Banda

Prefere links mais rápidos (fibra óptica > cabo de cobre > Wi-Fi). Usado pelo OSPF.

⏱️ Latência

Prefere caminhos com menor atraso. Muito importante para chamadas de voz e videochamadas.

🔒 Confiabilidade

Prefere links com menor histórico de falhas. Evita caminhos instáveis mesmo que mais rápidos.

Regra geral: entre duas rotas para o mesmo destino, o roteador sempre escolhe a mais específica (a que descreve melhor aquele destino) e, se empatarem, a de menor métrica.

Roteamento Estático

No roteamento estático, o administrador de rede configura as rotas manualmente, uma por uma.

✅ Vantagens

  • Simples de entender e configurar em redes pequenas
  • Previsível: o administrador controla exatamente o caminho
  • Sem consumo extra de banda (roteadores não trocam mensagens entre si)
  • Mais seguro: nenhum protocolo externo pode alterar as rotas

❌ Desvantagens

  • Não se adapta automaticamente a falhas
  • Se um link cair, o tráfego para até intervenção humana
  • Trabalhoso em redes grandes: cada rota precisa ser adicionada manualmente
  • Qualquer mudança na topologia exige reconfiguração manual
Analogia: mapa de papel impresso. Você planejou a rota antes de sair, mas se houver um bloqueio na estrada, o mapa não te ajuda — você precisa descobrir o desvio sozinho.

Roteamento Dinâmico

No roteamento dinâmico, os próprios roteadores trocam informações entre si e atualizam suas tabelas automaticamente.

✅ Vantagens

  • Adapta-se automaticamente a falhas — o tráfego desvia sem intervenção humana
  • Funciona bem em redes grandes e complexas
  • Configuração inicial mais trabalhosa, mas manutenção quase zero
  • Detecta novos caminhos assim que novos links são adicionados

❌ Desvantagens

  • Mais complexo de configurar e entender
  • Consome banda de rede para trocar mensagens de roteamento
  • Pode demorar alguns segundos para convergir após uma falha
  • Requer que os roteadores confiem uns nos outros
Analogia: GPS com trânsito em tempo real. Se houver um acidente na Marginal, ele detecta e recalcula o caminho automaticamente — você não precisa fazer nada.

Protocolos de Roteamento: Uma Visão Geral

Os protocolos de roteamento são as "línguas" que os roteadores usam para trocar informações sobre os caminhos disponíveis:

ProtocoloNome completoMétrica principalOnde é usado
RIP Routing Information Protocol Número de saltos Redes pequenas (raramente usado hoje)
OSPF Open Shortest Path First Largura de banda Redes corporativas
BGP Border Gateway Protocol Políticas e acordos A própria Internet — conecta provedores entre si
Não se preocupe em memorizar esses protocolos agora. O importante é entender que eles existem e qual é o papel de cada um. Veremos mais detalhes em aulas futuras.

O que é um Protocolo de Rede?

Definição: um protocolo de rede é um conjunto de regras e padrões que definem como dois dispositivos se comunicam — o formato das mensagens, a ordem dos eventos e como tratar erros.

🗣️ Analogia Humana

Para duas pessoas se comunicarem, precisam de:

  • Um idioma em comum (vocabulário + gramática)
  • Regras de conversação (quem fala quando)
  • Um canal (voz, escrita, gesto)
  • Forma de lidar com mal-entendidos (pedir para repetir)

Protocolos de rede fazem exatamente isso para dispositivos.

🌐 Protocolo na prática

Quando você acessa um site, vários protocolos entram em ação:

  • Seu navegador e o servidor "combinam" como trocar dados → HTTP
  • Os dados são divididos em pedaços e numerados → TCP
  • Cada pedaço recebe um endereço e é roteado → IP
  • Os pedaços viram sinais no cabo → Ethernet

Por que Precisamos de Vários Protocolos?

Cada camada da comunicação resolve um problema diferente. Cada protocolo faz uma coisa bem-feita — e eles trabalham juntos:

❓ Como o navegador pede uma página ao servidor? → HTTP / HTTPS
❓ Como garantir que todos os pedaços chegaram e na ordem certa? → TCP
❓ Como endereçar o destino de forma única no mundo inteiro? → IP
❓ Como transformar bits em sinais elétricos ou ondas de rádio? → Ethernet / Wi-Fi
Essa separação de responsabilidades é chamada de arquitetura em camadas. Vamos entendê-la com o Modelo OSI.

Camada 3 — Rede: o Papel do Roteador

A Camada 3 resolve o problema do endereçamento global e do roteamento entre redes diferentes. É aqui que entram o IP e os roteadores.

🔑 Responsabilidades da Camada 3

  • Atribuir um endereço lógico único a cada dispositivo na rede (endereço IP)
  • Dividir os dados em pacotes para transmissão
  • Determinar o melhor caminho de ponta a ponta
  • Encaminhar os pacotes de roteador em roteador até o destino

Camadas 4 a 7: Uma Prévia

As camadas superiores serão estudadas em detalhes nas próximas aulas. Veja o que cada uma faz:

4
Transporte (TCP/UDP): garante que os dados chegaram completos e na ordem certa (TCP) ou os envia sem confirmação, mais rápido (UDP). Aula 12.
5
Sessão: controla o diálogo entre aplicações — quem fala quando, como pausar e retomar uma conversa. Menos visível no dia a dia.
6

7

Encapsulamento: o que é?

Quando você envia dados pela rede, cada camada do Modelo OSI adiciona informações extras ao pacote — como colocar uma carta dentro de envelopes sucessivos.

📦 A Analogia dos Envelopes

Imagine que você vai enviar uma carta internacional:

  1. Escreve a carta (conteúdo = dados)
  2. Coloca num envelope com endereço → TCP
  3. Coloca numa caixa com endereço IP → IP
  4. A caixa vai num contêiner com rota de entrega → Ethernet
  5. O contêiner vai num navio/avião → Físico

📬 No receptor (desencapsulamento)

O processo é inverso — cada camada remove seu envelope:

  1. Camada Física: converte sinais em bits
  2. Ethernet: remove o cabeçalho de enlace
  3. IP: remove o cabeçalho de rede
  4. TCP: remove o cabeçalho de transporte
  5. Aplicação: lê os dados originais

Encapsulamento: Visualizando as Camadas

Veja como as camadas se empilham — cada uma envolve a anterior como um envelope:

Dados — conteúdo da aplicação (ex: página HTML, e-mail) Camada 7 — Aplicação
[Cabeçalho TCP]    Dados Camada 4 — Segmento TCP
[Cabeçalho IP]    Cabeçalho TCP    Dados Camada 3 — Pacote IP  ← o roteador lê aqui
[Cabeçalho Ethernet]    Cabeçalho IP    Cabeçalho TCP    Dados    [FCS] Camada 2 — Frame Ethernet
10110100 01001010 11001011...  (bits elétricos / luz / ondas de rádio) Camada 1 — Físico

O roteador abre o pacote até a Camada 3, lê o IP de destino, e remonta o frame para a próxima rede — trocando os cabeçalhos de Camada 2.

O Protocolo IP: o que ele faz?

O IP (Internet Protocol) é o protocolo da Camada 3 responsável por endereçar e encaminhar pacotes entre redes.

⚙️ Responsabilidades do IP

  • Endereçar — atribui a cada dispositivo um endereço único (o IP)
  • Fragmentar — divide dados grandes em pacotes menores que a rede consegue transportar
  • Rotear — define como roteadores encaminham esses pacotes até o destino

⚠️ O que o IP NÃO faz

  • Não garante entrega — um pacote pode ser perdido e o IP não avisa
  • Não garante ordem — pacotes do mesmo arquivo podem chegar fora de sequência
  • Não verifica o conteúdo — só checa a integridade do cabeçalho
Quem garante a entrega confiável é o TCP (Camada 4). O IP apenas faz o melhor esforço (best effort).

Cada pacote IP carrega um cabeçalho com campos que os roteadores usam para tomar decisões:

CampoO que armazena?Exemplo

O roteador lê principalmente o IP de Destino e o TTL para tomar sua decisão. O restante do cabeçalho é ignorado por ele.

A Jornada de um Pacote: o Cenário

Vamos acompanhar um pacote do início ao fim. Cenário: você acessa google.com do seu computador em casa.

1
Resolução de nome: seu navegador pergunta ao servidor DNS "qual é o IP do google.com?" e recebe a resposta. (DNS = Aula 13)
2
Montagem do pacote: seu PC cria o pacote IP com IP de origem = seu IP e IP de destino = IP do Google.
3
Decisão local: "O Google está na minha rede local?" → Não. → O pacote é enviado para o gateway padrão (seu roteador Wi-Fi).
4
Roteamento em cadeia: o roteador doméstico consulta sua tabela → encaminha ao ISP → backbone → datacenter do Google.
